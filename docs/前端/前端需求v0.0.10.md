# 前端需求文档 v0.0.10（基于 v0.0.9 增量）

## 1. 文档信息
- 版本：`v0.0.10`
- 基线日期：`2026-03-18`
- 上一版本：`/Users/lingyunxieqing/Desktop/gos/docs/前端/前端需求v0.0.9.md`
- 本版主题：`ArgoCD/GitOps 变量替换前置收口 + project_name 单值场景`

## 2. 本版目标
在 `v0.0.8` 的 `ArgoCD/GitOps` 管理能力和 `v0.0.9` 的回滚、取值进度基础上，前端继续收口一条更可执行的规则：
- 当模板的 `CD` 走 `ArgoCD` 时，`project_name` 先只支持单值场景
- 用户在创建发布单时，不再把 `project_name` 当成“可多选的微服务清单”去理解
- 页面要把这条规则明确提示出来，为下一版 GitOps YAML 变量替换做前置约束

## 3. 范围
包含：
- 发布模板页 `CD=ArgoCD` 的说明文案增强
- 新建发布单页 `project_name` 单值交互
- 新建发布单页 `project_name` 多值拦截提示
- `GitOps` 提交信息模版继续按标准平台 Key 选取字段

不包含：
- GitOps 任意 YAML 路径可视化编辑
- `project_name` 多值批量写回 YAML
- 多服务并行 ArgoCD CD 编排
- GitOps 替换规则设计页

## 4. 背景
当前 `CD=ArgoCD` 的平台执行逻辑仍以：
- 基于标准平台 Key 取值
- 修改 GitOps 仓库
- 触发 ArgoCD Sync
为主。

后续如果需要支持：
- `Deployment.metadata.name`
- `ConfigMap.metadata.name`
- 镜像名中的服务名段
- 其他 YAML 字段

则 `project_name` 必须先从“多选服务列表”收口成“单值服务标识”，否则前端和后端都无法稳定判断本次应该修改哪一组资源。

## 6. 新建发布单页
文件目标：
- `/Users/lingyunxieqing/Desktop/gos/frontend/src/views/release/ReleaseOrderCreateView.vue`

### 6.1 总体规则
当模板满足：
- 已启用 `CI`
- `CD=ArgoCD`

且 `CI` 参数中存在：
- `param_key = project_name`

则前端对该字段启用“单值场景”规则。

### 6.2 交互要求
若 `project_name` 是 `choice` 类型：
- 即使执行器原始元数据提示支持多选
- 页面也按单选交互展示
- 占位文案明确提示：`当前按单值处理，请只选择一个项目`

若 `project_name` 是输入框类型：
- 占位文案明确提示：`当前按单值处理，请输入单个项目名`
- 字段下方增加辅助说明，提示当前模板的 `CD` 使用 `ArgoCD`，`project_name` 先按单值场景处理

### 6.3 提交前校验
提交创建发布单时，如果 `project_name` 值中包含明显多值格式：
- 英文逗号 `,`
- 中文逗号 `，`
- 换行

则前端直接拦截，并提示：
- `当前模板的 CD 使用 ArgoCD，project_name 仅支持单值，请只选择一个项目`

### 6.4 页面说明文案
`CD=ArgoCD` 的说明文案中，需要额外补充：
- 当前版本如果使用 `project_name`，会按单值场景处理

## 7. GitOps 管理页
文件目标：
- `/Users/lingyunxieqing/Desktop/gos/frontend/src/views/component/GitOpsManagementView.vue`

### 7.1 提交信息模版
延续当前规则：
- 名称统一为 `提交信息模版`
- 字段来源于标准平台 Key
- 用户从字段列表选择并插入 `{param_key}`
- 发布执行时再从流程参数中取值

### 7.2 GitOps 替换规则配置
当模板 `CD` 留空走 `ArgoCD` 时，模板页新增 `GitOps 替换规则` 区块：
- 左侧来源字段：
  - 优先展示 `CI` 中已勾选的标准平台 Key
  - 同时补充系统内置字段
- 右侧目标字段：
  - 来自 GitOps 仓库扫描出的 YAML 标量字段
  - 基于应用 `app_key` 扫描 `apps/<app_key>/overlays/<env>` 下的 YAML
- 每条规则支持配置：
  - 标准字段
  - YAML 字段
  - 取值模版

### 7.3 取值模版
每条 GitOps 替换规则都支持一个 `取值模版`：
- 默认值为 `{标准Key}`
- 可按需要调整为：
  - `{project_name}`
  - `{project_name}-config`
  - `{env}-{project_name}`

取值模版的占位符仍来自标准平台 Key，由发布流程实际取值时再渲染。
