- 当前状态：`已落地并持续迭代`

## 2. 本版目标
在当前单 ArgoCD 页面基础上，前端需要支持：
- 管理多个 ArgoCD 实例
- 管理环境到 ArgoCD 实例的绑定关系
- 管理 GitOps 实例，并在 ArgoCD 实例中选择关联的 GitOps 实例
- 发布模板编辑时根据环境自动绑定 ArgoCD 执行器
- 在发布详情中展示“本次实际命中的 ArgoCD 实例”
- 在“我的应用”页为所有已登录用户展示发布流程介绍

## 3. 范围
包含：
- ArgoCD 实例管理页
- GitOps 实例管理页
- 环境绑定配置页
- 发布模板中 ArgoCD 自动绑定提示
- 发布详情中实例命中结果展示
- “我的应用”页发布流程介绍抽屉

不包含：
- 可视化创建 ArgoCD Project
- 集群资源拓扑图
- 多集群流量编排界面

## 4. 背景
当前多实例能力已经落地，但前端还需要把关系说明与交互边界讲清楚：
- 用户需要知道平台支持多 ArgoCD、多 GitOps
- 用户需要知道模板里的 ArgoCD 是按环境自动解析的
- 用户需要知道 Git 仓库并不是在发布时手工选择，而是通过 ArgoCD / GitOps 实例链路自动关联

因此前端必须把以下信息展示清楚：
- 有哪些 ArgoCD 实例
- 每个 ArgoCD 实例关联哪个 GitOps 实例
- 每个实例对应哪个环境/集群
- 当前模板在不同环境下会命中哪个 ArgoCD
- 某张发布单最终用了哪个 ArgoCD

## 8. 发布模板页
文件目标：
- `/Users/lingyunxieqing/Desktop/gos/frontend/src/views/release/ReleaseTemplateView.vue`

### 8.1 ArgoCD 选择方式调整
当 `CD` 走 ArgoCD 时：
- 不再让用户选择某个具体 ArgoCD 实例
- 改为显示说明：
  - 平台会根据发布环境自动绑定 ArgoCD 执行器

当前模板页还新增了显式 `CD 类型` 切换：
- `管线`
- `ArgoCD`

交互规则：
- 选择 `管线` 时，只显示 `CD 绑定管线`
- 选择 `ArgoCD` 时，只显示 ArgoCD / GitOps 相关配置

### 8.2 页面提示
模板页需要明确提示：
- `dev/test/prod` 将根据系统中的环境绑定规则自动命中对应集群
- 若环境尚未绑定 ArgoCD 实例，模板保存会失败或给出明确提示

### 8.3 规则展示
建议在模板页显示只读提示区：
- `dev -> 开发集群 ArgoCD`
- `test -> 测试集群 ArgoCD`
- `prod -> 生产集群 ArgoCD`

让用户知道模板并不是“没有绑定”，而是“自动绑定”。

## 14. 结论
`v0.0.12` 前端的关键不是让用户手工多选更多配置，而是把“多集群复杂度”收进系统配置里，把模板和发布单体验尽量做轻。

目标体验应当是：
- 管理员在组件管理里维护好多套 ArgoCD 和环境映射
- 普通发布用户只需要选环境
- 平台自动把发布送到正确的 ArgoCD / K8s 集群
