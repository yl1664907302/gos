- 当前状态：`规划中`

## 2. 阶段目标
阶段一不再继续扩展发布方式，而是优先把平台从“能发”提升到“稳发、可控、可追”。

本阶段前端需要承接四类核心能力：
- 环境当前版本台账
- 发布前预检
- 并发发布控制与环境锁
- 通知结果回显

## 3. 本版范围
### 3.1 纳入范围
- 我的应用页增加环境当前版本视图
- 发布单新建/执行前增加预检结果展示
- 发布单列表/详情增加环境锁与并发限制提示
- 系统管理新增并发发布配置页面
- 发布结果增加通知投递结果展示

### 3.2 暂不纳入
- 审批流
- 时间窗/冻结期
- 自动回滚
- 策略发布（蓝绿、金丝雀、分批）
- 发布大盘与统计分析

## 4. 业务问题
当前平台存在以下体验缺口：
- 用户无法从平台直观看到某个应用某个环境当前实际版本。
- 用户点击发布后，通常要等执行失败才知道前置条件不满足。
- 同一应用/环境存在并发点击发布的风险，前端缺少明确拦截和提示。
- 发布失败、回滚失败虽然有页面状态，但通知结果和投递是否成功不可见。

## 5.2 发布单新建页
文件目标：
- `/Users/lingyunxieqing/Desktop/gos/frontend/src/views/release/ReleaseOrderCreateView.vue`

新增能力：
- 在用户选择完应用、模板、环境后，支持触发预检。
- 预检结果需结构化展示，至少包括：
  - 模板是否完整

- 若预检通过：
  - 展示绿色通过状态
  - 支持继续执行

预检结果建议分组：
- `模板检查`
- `执行器检查`
- `环境检查`
- `并发检查`

## 6. 关键交互规范
## 6.1 并发发布提示
当用户点击发布时，如果环境已被锁定或已有运行中发布：
- 不应只弹出通用错误 toast
- 应展示明确原因：
  - `当前环境已有运行中发布`
  - `当前环境被锁定`
  - `当前仓库分支处于占用中`
- 同时给出可理解的下一步建议：
  - 查看运行中发布单
  - 等待锁释放
  - 联系管理员

## 6.2 预检展示
预检结果不应只是一条“预检失败”的文案，必须是逐项结果：
- 通过项：绿色状态
- 警告项：黄色状态
- 阻断项：红色状态

## 6.3 当前版本展示
环境当前版本建议优先展示：
- `image_version`
- 若无镜像版本，则显示：
  - `git_ref`
  - 或 `git_sha`

同时展示：
- 最近一次成功发布时间
- 最近一次成功单号

## 6.4 通知结果展示
通知结果需至少展示：
- 渠道名称
- 投递时间
- 投递结果
- 错误信息（若失败）

## 7. 新增状态与标签建议
前端需要新增并统一这批标签：
- `预检通过`
- `预检失败`
- `环境锁定`
- `等待锁释放`
- `并发受限`
- `通知成功`
- `通知失败`
- `当前版本`

## 8. 数据结构需求（前端侧）
前端需要准备接入以下新结构：

### 8.1 应用工作台环境摘要
每个应用下的环境条目建议包含：
- `env_code`
- `current_version`
- `current_version_type`
- `current_release_order_no`
- `current_release_time`
- `health_status`
- `lock_status`
- `lock_reason`
- `running_order_no`

### 8.2 预检结果
建议结构：
- `summary_status`
- `items[]`
- `items[].code`
- `items[].title`
- `items[].status`
- `items[].message`
- `items[].blocking`

### 8.3 并发配置
建议结构：
- `enabled`
- `lock_scope`
- `conflict_strategy`
- `lock_timeout_sec`
- `allow_admin_override`

### 8.4 通知结果
建议结构：
- `channel`
- `status`
- `message`
- `sent_at`

## 9. 页面风格要求
本版仍延续现有 UI 规范：
- 白色主内容面板
- 深石板蓝壳层
- 驾驶舱式重点状态卡
- 预检/锁状态/通知结果使用明亮提示面板，不允许回退为暗色说明块

## 11. 结论
`v0.0.15` 前端阶段一的核心目标不是增加更多发布方式，而是补齐“可靠发布”的可见性与控制能力：
- 用户先看清当前环境版本
- 用户先知道能不能发
- 用户知道为什么不能发
- 用户知道发完有没有通知出去
