## 2. 版本目标
当前平台的发布单状态更偏“执行状态”，前端需要大量依赖推导才能解释：
- 这张单是不是还在排队
- 这张单是不是还没审批
- 这张单是审批通过但未执行，还是已经开始执行

`v0.0.16` 的目标是把发布单前端展示，从“执行态视图”升级为“业务态视图”。

## 3. 现状问题
当前主状态只有：
- `pending`
- `running`
- `success`
- `failed`
- `cancelled`

缺口：
- 没有 `草稿`
- 没有 `待审批 / 审批中 / 审批通过 / 审批拒绝`
- 没有独立的 `排队中`
- `success / failed` 无法区分“审批成功”和“部署成功”
- 列表页、详情页、通知文案都需要额外推导，用户理解成本高

## 4. 目标状态机
前端按以下业务态展示并交互：

- `draft`：草稿
- `pending_approval`：待审批
- `approving`：审批中
- `approved`：审批通过
- `rejected`：审批拒绝
- `queued`：排队中
- `deploying`：发布中
- `deploy_success`：发布成功
- `deploy_failed`：发布失败
- `cancelled`：已取消

说明：
- `approved` 表示业务上允许执行，但还未进入真正部署。
- `queued` 表示已通过预检/审批，但因并发锁、批次顺序或队列策略而等待。
- `deploying` 才表示真正进入执行链路。

## 6. 前端展示规范
## 6.1 状态标签
推荐统一标签文案：
- `草稿`
- `待审批`
- `审批中`
- `审批通过`
- `审批拒绝`
- `排队中`
- `发布中`
- `发布成功`
- `发布失败`
- `已取消`

## 6.2 状态颜色
建议统一：
- `draft`：中性灰
- `pending_approval / approving`：暖黄
- `approved`：青蓝
- `queued`：蓝灰
- `deploying`：亮蓝
- `deploy_success`：亮绿
- `deploy_failed`：红色
- `rejected`：红棕
- `cancelled`：灰黑

## 6.3 动效
- `approving` 使用轻量旋转或呼吸动效
- `queued` 使用轻量等待动效，不得与 `deploying` 共用完全相同的动效
- `deploying` 继续保留当前执行中动态图标，但要比 `queued` 更积极

## 7. 按钮与权限规则
前端按钮应按目标状态控制：

- `draft`
  - 可编辑
  - 可删除
  - 可提交审批（后续）
  - 可直接发布（若当前模板无需审批）
- `pending_approval`
  - 不可发布
  - 可取消
- `approving`
  - 不可发布
  - 不可重复提交审批
- `approved`
  - 可发布
- `queued`
  - 不可重复发布
  - 可取消
- `deploying`
  - 不可重复发布
  - 可取消（若后端支持）
- `deploy_success`
  - 可显示 `回滚到此版本`
- `deploy_failed`
  - 可根据模板模型显示：
    - 重试
    - 回滚
    - 复制新单
- `rejected`
  - 可复制/编辑后重新发起
- `cancelled`
  - 不可直接再次执行，建议复制新单

## 8. 兼容映射要求
在后端未完全切到新状态机前，前端需要兼容老状态映射：

- `pending` -> `approved`
- `running + waiting_for_lock` -> `queued`
- `running + has_running_execution` -> `deploying`
- `success` -> `deploy_success`
- `failed` -> `deploy_failed`
- `cancelled` -> `cancelled`

说明：
- 这是前端兼容阶段的临时映射，不应长期作为最终方案。

## 9. 前端数据结构调整建议
发布单基础结构建议新增：
- `business_status`
- `approval_status`
- `queue_position`
- `queued_by_order_no`
- `queued_reason`
- `can_execute`
- `can_cancel`
- `can_rollback`

并发批次结构建议补充：
- `queue_state`
- `queue_position`
- `blocking_order_no`

## 10. 验收标准
前端完成后，需要满足：
1. 用户能一眼分辨“排队中”和“发布中”。
2. 用户能一眼分辨“审批通过但未执行”和“发布成功”。
3. 发布列表、详情页、我的应用页三处状态语义一致。
4. 并发批次中后续单不能再显示为失败，必须明确显示排队态。
5. 旧状态迁移期间，前端页面不能出现文案冲突或同一单多处状态不一致。
