## 2. 版本目标
在保留 v0.0.2 已有“我的应用”“管线绑定”“Jenkins管理”能力不变的基础上，新增：
- `应用管理 > 标准字库` 页面，展示并维护平台标准字段库
- `组件管理 > 执行器参数` 页面，展示指定应用已绑定的 Jenkins 管线真实参数
- 支持将 `executor_param_def.param_key` 通过下拉选择方式映射到 `platform_param_dict.param_key`

本版核心目标：
- 前端可维护平台标准字段字典
- 前端可基于“应用”维度查看其已绑定 Jenkins 管线的真实参数
- 前端可维护真实参数与平台标准字段之间的映射关系

## 3. 范围与边界
包含功能：
- 标准字库列表、详情、新增、编辑、删除
- 标准字库 `param_key` 小写校验

- 执行器参数详情查看
- 执行器参数“平台标准参数 key”编辑
- 平台标准字段下拉填充

不包含功能：
- 执行器参数手动新增
- 执行器参数手动删除
- Jenkins 参数手动同步按钮默认开放给普通业务用户
- Jenkins / ArgoCD 执行触发
- 发布单据渲染与发布流程表单

## 6. 功能需求

### 6.1 标准字库页
页面目标：
- 展示 `platform_param_dict` 内容
- 支持完整 CRUD
- 作为“执行器参数”模块的标准字段下拉数据源

#### 6.1.1 查询区域
筛选条件：
- `param_key`
- `name`
- `status`（全部 / 启用 / 停用）
- `builtin`（全部 / 是 / 否）

交互要求：
- 支持查询
- 支持重置
- 支持分页联动

#### 6.1.2 列表区域
表格字段：
- `param_key`
- `name`
- `description`
- `param_type`
- `required`
- `builtin`
- `status`
- `updated_at`

操作列：
- `查看`
- `编辑`
- `删除`

顶部操作：
- `新增标准字段`

#### 6.1.3 新增标准字段
展示方式建议：
- `a-modal` 表单

表单字段：
- `param_key`（必填）
- `name`（必填）
- `description`（选填）
- `param_type`（必填，下拉：`string | choice | bool | number`）
- `required`（必填，开关）
- `builtin`（必填，开关）
- `status`（必填，下拉：`启用=1 / 停用=0`）

前端校验：
- `param_key` 必填
- `param_key` 必须小写
- `param_key` 建议正则校验：`^[a-z][a-z0-9_]*$`
- 可在输入时自动转小写，但提交前仍需校验

提交成功：
- 关闭弹窗
- 成功提示
- 刷新列表

#### 6.1.4 查看详情
- 调用详情接口
- 使用抽屉或只读弹窗展示完整字段

#### 6.1.5 编辑标准字段
- 进入编辑前先拉详情
- 字段与新增保持一致
- `param_key` 仍然允许编辑，但必须满足小写规则

#### 6.1.6 删除标准字段
- 点击删除后进行二次确认
- 若后端返回“内置字段不可删除”或“已被引用不可删除”，前端直接展示后端错误信息

### 6.2 执行器参数页
页面目标：
- 基于“应用”维度展示其已绑定 Jenkins 管线的真实参数
- 允许用户维护每个真实参数对应的平台标准字段映射

#### 6.2.1 页面交互模型
顶部查询区域必须包含：
- `application_id` / `application_name` 选择器（必选）
- `binding_type`（默认 `ci`，可选 `ci | cd`）
- `param_key`
- `visible`
- `editable`

交互规则：
- 未选择应用前，列表不加载，展示空状态“请先选择应用”
- 选择应用后，调用按应用查询接口加载数据
- 若应用未绑定对应类型的 Jenkins 管线，展示后端错误信息或空状态提示
- 当前页面展示的是“指定应用绑定的 Jenkins 执行器参数”，不是全量组件参数总表

#### 6.2.2 应用选择器
建议方案：
- 使用远程搜索下拉
- 下拉展示文案：`name (key)`
- 选中值：`application.id`

数据来源：
- `GET /applications`

#### 6.2.3 参数列表
表格字段建议：
- `executor_param_name`
- `param_key`
- `param_type`
- `required`
- `default_value`
- `description`
- `visible`
- `editable`
- `source_from`
- `sort_no`
- `updated_at`

操作列：
- `查看`
- `编辑映射`

说明：
- 本页不提供“新增参数”“删除参数”
- 本页不允许编辑 `executor_param_name`、`param_type`、`required`、`default_value` 等 Jenkins 主数据字段

#### 6.2.4 编辑映射
展示方式建议：
- 行内编辑或弹窗编辑均可

仅允许编辑字段：
- `param_key`

编辑方式：
- `param_key` 使用下拉选择
- 下拉数据来源于“标准字库”
- 下拉展示建议：`name (param_key)`
- 下拉值为：`param_key`
- 支持清空，清空表示取消映射

数据限制：
- 建议只加载 `status=1` 的标准字段作为可选项
- 若当前记录已绑定了一个已停用字段，编辑态仍需能回显当前值

提交成功：
- 成功提示
- 刷新当前列表

#### 6.2.5 参数详情
- 支持查看完整字段
- `raw_meta` 若内容较长，可放在折叠区域或代码块中展示

### 6.3 与现有模块关系
- “标准字库”是“执行器参数”映射编辑的数据来源
- “管线绑定”用于决定某应用当前绑定了哪条 Jenkins 管线
- “执行器参数”通过应用维度去读取该绑定关系对应的 Jenkins 参数数据
- “Jenkins管理”继续作为 Jenkins 管线主数据的只读展示页

## 7. API 对接要求

### 7.1 标准字库列表
- 方法: `GET /platform-param-dicts`
- Query:
- `param_key?: string`
- `name?: string`
- `status?: 1 | 0`
- `builtin?: boolean`
- `page?: number`
- `page_size?: number`
- 响应:
- `data: PlatformParamDict[]`
- `page`
- `page_size`
- `total`

### 7.2 标准字库新增
- 方法: `POST /platform-param-dicts`
- Body:
- `param_key`
- `name`
- `description`
- `param_type`
- `required`
- `builtin`
- `status`
- 响应: `{ data: PlatformParamDict }`

### 7.3 标准字库详情
- 方法: `GET /platform-param-dicts/{id}`
- 响应: `{ data: PlatformParamDict }`

### 7.4 标准字库更新
- 方法: `PUT /platform-param-dicts/{id}`
- Body:
- `param_key`
- `name`
- `description`
- `param_type`
- `required`
- `builtin`
- `status`
- 响应: `{ data: PlatformParamDict }`

### 7.5 标准字库删除
- 方法: `DELETE /platform-param-dicts/{id}`
- 响应: `204 No Content`

### 7.6 执行器参数列表（按应用）
- 方法: `GET /applications/{id}/executor-param-defs`
- Query:
- `binding_type?: ci | cd`
- `visible?: boolean`
- `editable?: boolean`
- `param_key?: string`
- `page?: number`
- `page_size?: number`
- 响应:
- `data: ExecutorParamDef[]`
- `page`
- `page_size`
- `total`

说明：
- 默认 `binding_type=ci`
- 后端仅返回该应用已绑定的 `provider=jenkins` 的执行器参数

### 7.7 执行器参数详情
- 方法: `GET /executor-param-defs/{id}`
- 响应: `{ data: ExecutorParamDef }`

### 7.8 执行器参数映射更新
- 方法: `PUT /executor-param-defs/{id}`
- Body:
- `param_key`
- 响应: `{ data: ExecutorParamDef }`

说明：
- `param_key` 允许传空字符串，表示取消映射
- `param_key` 非空时必须来自标准字库

### 7.9 应用选择器数据
- 方法: `GET /applications`
- 建议 Query:
- `page=1`
- `page_size=100`
- 可按需加 `name` 或 `key` 搜索

### 7.10 标准字段下拉数据
- 方法: `GET /platform-param-dicts`
- 建议 Query:
- `status=1`
- `page=1`
- `page_size=100`

## 8. 错误处理与提示文案
- `400`
- 标准字库表单校验失败
- `param_key` 非小写
- 执行器参数请求中 `binding_type` 非法
- 提示：直接展示后端 `error`

- `404`
- 应用不存在
- 标准字段不存在
- 应用未绑定对应类型 Jenkins 管线
- 执行器参数记录不存在
- 提示：`未找到对应数据`

- `409`
- 标准字段 `param_key` 冲突
- 内置字段不可删除
- 标准字段已被引用不可删除/修改
- 提示：直接展示后端 `error`

- `500`
- 提示：`系统异常，请稍后重试`
