- 本版主题：`发布执行可观测性与交互稳定性增强`

## 2. 版本目标
在 `v0.0.4` 发布单模块基础上，完善“可执行、可观测、可取消、少手动刷新”的前端体验，重点覆盖：
- 发布单列表与详情自动刷新
- 发布详情页单次发布入口
- Jenkins 执行日志流式展示
- 取消发布后的状态联动展示
- 创建发布单时 `son_service` 与参数联动增强

## 3. 本版增量范围
包含：
- 发布单列表页自动刷新与发布预审
- 发布详情页执行按钮、状态动画、日志面板
- 创建发布单 `son_service` 字段与 `project_name` 参数联动
- 发布参数单选/多选能力适配（基于后端 `single_select`）

不包含：
- WebSocket 日志能力（当前为 SSE）
- 多窗口状态同步控制
- 发布审批与回滚交互流程

## 4. 当前实现进度（2026-03-13）

### 4.1 发布单列表页（`/releases`）
- 已完成查询、分页、状态标签、详情/取消操作
- 已完成“发布”预审弹窗（展示基础信息 + 参数快照）
- 确认发布后调用：`POST /release-orders/{id}/execute`
- 已支持自动刷新（默认每 `5s`），并在以下场景暂停静默刷新：
- 页面不可见
- 预审弹窗打开或提交中
- `running` 状态已使用统一转圈动效展示

### 4.2 发布单详情页（`/releases/:id`）
- 已新增详情页“发布”按钮，且只允许单次点击触发（前端锁）
- 仅 `pending` 状态可点击发布，`pending/running` 可取消
- 已支持自动刷新（默认每 `5s`，用于状态与步骤更新）
- 已支持执行步骤与主状态的统一加载动效（`running` 转圈）
- 已新增“构建日志”区域：
- SSE 连接后端日志流接口：`GET /release-orders/{id}/logs/stream`
- 支持状态提示、日志追加、断线重连、清空日志

### 4.3 创建发布单页（`/releases/new`）
- 已支持新增字段：`son_service`（非必填）
- `son_service` 已与绑定管线中的 `project_name` 参数候选值联动下拉
- `git_ref` 已与平台标准 Key=`branch` 参数候选值联动下拉
- 非必填发布参数允许留空提交
- 已适配参数类型策略：
- `single_select=true` 使用单选
- 其余多值参数默认按多选处理

### 4.4 API 对接进度
- 已对接发布单核心接口：
- `GET /release-orders`
- `POST /release-orders`
- `GET /release-orders/{id}`
- `POST /release-orders/{id}/execute`
- `POST /release-orders/{id}/cancel`
- `GET /release-orders/{id}/params`
- `GET /release-orders/{id}/steps`
- 已新增日志流 URL 构建能力：
- `buildReleaseOrderLogStreamURL(id, start)`

## 5. 已验证项
- 已执行：`npm run build`（通过）
- 已验证：列表/创建/详情路由可达，发布预审与执行请求可发起
- 已验证：详情页日志面板可连接流并持续追加日志内容

## 6. 当前限制与待完善
- 日志面板当前未做大日志量虚拟滚动与检索
- SSE 断线重连后为增量续传，暂不支持完整历史重放
- 前端不直接控制 Jenkins 队列，仅通过后端状态回写展示最终结果
