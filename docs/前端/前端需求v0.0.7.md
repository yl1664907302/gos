# Vue3 + Ant Design Vue 前端需求文档 v0.0.7

## 1. 文档信息
- 版本：`v0.0.7`
- 对应后端基线日期：`2026-03-16`
- 上一版本：`/Users/lingyunxieqing/Desktop/gos/docs/前端/前端需求v0.0.6.md`
- 本版主题：`发布模板支持 CI/CD 双管线 + 新建发布单双参数视图 + 发布详情双日志/双进度`

## 2. 版本目标
在现有发布模板与发布单页面基础上，升级为多执行单元发布体验，实现：
- 一个发布模板可同时配置 `CI` 与 `CD`
- 新建发布单时可分别填写 `CI 参数` 与 `CD 参数`
- 发布详情页可分别展示：
- `CI 参数快照`
- `CD 参数快照`
- `CI 日志`
- `CD 日志`
- `CI 管线进度`
- `CD 管线进度`
- 若应用无 `CD` 绑定或模板未启用 `CD`，相关模块不展示

## 3. 范围与边界
包含：
- 发布模板页面升级
- 新建发布单页面升级
- 发布详情页面升级
- 按 `scope` 的日志与进度展示

不包含：
- 审批流页面
- 多环境批量发布页面
- ArgoCD 独立控制台页面

## 4. 菜单与路由
沿用现有菜单结构，不新增一级菜单。

保留：
- `发布管理 -> 发布模板`
- `发布管理 -> 发布单`

路由沿用：
- `/release-templates`
- `/releases`
- `/releases/new`
- `/releases/:id`

## 5. 页面设计

## 5.1 发布模板页升级

### 5.1.1 页面目标
让一个模板可以同时配置：
- `CI`
- `CD`

### 5.1.2 页面结构
- 模板基础信息卡片
- `CI 配置` 卡片
- `CD 配置` 卡片
- 底部保存区

### 5.1.3 模板基础信息
字段：
- 模板名称
- 所属应用
- 状态
- 备注

### 5.1.4 CI 配置卡片
仅支持 Jenkins。

字段：
- 是否启用 `CI`
- `CI 绑定管线`
- `CI 参数选择区`

参数选择规则：
- 数据来源于所选 `CI` 绑定对应的真实管线参数
- 仅可选择已完成平台标准 Key 映射的参数
- 展示名称使用平台标准字段名称

### 5.1.5 CD 配置卡片
支持：
- Jenkins
- ArgoCD

字段：
- 是否启用 `CD`
- `CD 绑定管线`
- `CD 参数选择区`

参数选择规则：
- 数据来源于所选 `CD` 绑定对应的真实管线参数
- 同样仅允许选择已完成平台标准 Key 映射的参数

### 5.1.6 模板页交互要求
- `CI/CD` 两块独立展示，不混排
- 未启用某个 scope 时，对应卡片内容折叠或禁用
- 切换绑定后，参数源应重新加载
- 参数排序支持按模板内顺序保存

## 5.2 新建发布单页升级

### 5.2.1 页面目标
改成“先选模板，再按模板展示参数”。

### 5.2.2 基础字段
基础字段建议收敛为：
- 应用
- 发布模板
- 备注
- 创建者（只读展示）

说明：
- 不再手动选择 `CI/CD` 管线
- 管线由模板决定

### 5.2.3 参数视图
页面按模板自动展示：
- `CI 参数`
- `CD 参数`

展示规则：
- 若模板仅启用 `CI`，只展示 `CI 参数`
- 若模板仅启用 `CD`，只展示 `CD 参数`
- 若模板同时启用 `CI + CD`，展示两个独立卡片

### 5.2.4 参数字段要求
- 与基础字段保持相同的表单风格
- 每个 scope 内部使用独立表单布局
- 参数标题使用平台标准字段名称
- 参数录入控件按后端元数据决定：
- 文本输入
- 下拉选择
- 单选
- 布尔开关

### 5.2.5 参数视图要求
- `CI 参数` 与 `CD 参数` 必须使用不同标题与卡片分组
- 禁止把 `CI/CD` 参数混在同一个列表里
- 加载参数时需有单独 loading 态

## 5.3 发布详情页升级

### 5.3.1 基础信息区
继续展示：
- 发布单号
- 应用名称
- 模板名称
- 创建者
- 状态
- 备注
- 创建时间
- 更新时间

### 5.3.2 参数快照区
参数快照拆成：
- `CI 参数快照`
- `CD 参数快照`

规则：
- 应用无 `CD` 或本次发布无 `CD` 执行单元时，不展示 `CD 参数快照`

### 5.3.3 执行步骤区
执行步骤按 scope 展示：
- `全局步骤`
- `CI 步骤`
- `CD 步骤`

至少需要让用户明显感知：
- 当前执行到 `CI` 还是 `CD`
- `CD` 是否已开始
- `CD` 是否因为 `CI` 失败而未执行

### 5.3.4 日志区
日志区拆成：
- `CI 日志`
- `CD 日志`

规则：
- `CI` 为 Jenkins 时展示 Jenkins 构建日志
- `CD` 为 Jenkins 时展示 Jenkins 构建日志
- `CD` 为 ArgoCD 时展示部署状态日志/事件流
- 无 `CD` 时，不显示 `CD 日志`

交互要求：
- 两块日志区相互独立
- 各自拥有刷新/重连/跟随日志状态
- 不可因为查看 `CI` 日志而打断 `CD` 日志

### 5.3.5 管线进度区
管线进度拆成：
- `CI 管线进度`
- `CD 管线进度`

规则：
- 仅 Jenkins scope 展示真实 stage 进度
- ArgoCD scope 展示部署状态进度卡片
- 无 `CD` 时，不显示 `CD 管线进度`

## 6. 前端数据模型

### 6.1 发布模板
模板详情建议返回：
- `template`
- `bindings`
- `params`

其中 `bindings` 至少包含：
- `pipeline_scope`
- `binding_id`
- `binding_name`
- `provider`
- `pipeline_id`
- `enabled`

### 6.2 发布单详情
详情建议返回：
- `order`
- `executions`
- `params`
- `steps`
- `pipeline_stages`

其中每一项都应带：
- `pipeline_scope`

## 7. 接口依赖

### 7.1 发布模板
- `GET /release-templates`
- `GET /release-templates/{id}`
- `POST /release-templates`
- `PUT /release-templates/{id}`
- `DELETE /release-templates/{id}`
- `GET /applications/{id}/release-template-sources`

### 7.2 发布单
- `POST /release-orders`
- `GET /release-orders`
- `GET /release-orders/{id}`
- `GET /release-orders/{id}/params`
- `GET /release-orders/{id}/steps`
- `GET /release-orders/{id}/executions`

### 7.3 日志与进度
- `GET /release-orders/{id}/logs/stream?scope=ci`
- `GET /release-orders/{id}/logs/stream?scope=cd`
- `GET /release-orders/{id}/pipeline-stages?scope=ci`
- `GET /release-orders/{id}/pipeline-stages?scope=cd`
- `GET /release-orders/{id}/pipeline-stages/{stage_id}/log`

## 8. 权限与展示规则
- 继续沿用现有应用级权限模型
- 用户能查看该应用，才能查看该应用下模板与发布单
- 用户能发布该应用，才能创建包含 `CI/CD` 的发布单
- 参数快照展示权限仍受“参数快照查看权限”控制

## 9. 交互与 UX 约束
- `CI/CD` 卡片必须风格一致，但标题明确区分
- 无 `CD` 时，不允许展示空白的 `CD` 区块
- 日志与进度应尽量静默刷新，避免频繁闪烁
- `CI` 成功后 `CD` 才开始时，页面应体现等待状态，而不是直接空白
- 若 `CI` 失败导致 `CD` 未执行，应有清晰状态文案

## 10. 验收标准
- 发布模板页可同时配置 `CI + CD`
- 新建发布单页可分别展示与填写 `CI/CD` 参数
- 发布详情页可分别展示 `CI/CD` 参数快照
- 发布详情页可分别展示 `CI/CD` 日志
- 发布详情页可分别展示 `CI/CD` 管线进度
- 无 `CD` 时，页面不展示 `CD` 相关模块

- M3：发布详情页双日志/双进度改造
