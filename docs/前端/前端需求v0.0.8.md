# 前端需求文档 v0.0.8（基于 v0.0.7 增量）

## 1. 文档信息
- 版本：`v0.0.8`
- 基线日期：`2026-03-17`
- 上一版本：`/Users/lingyunxieqing/Desktop/gos/docs/前端/前端需求v0.0.7.md`
- 本版主题：`ArgoCD 管理模块 + ArgoCD CD 发布视图初版`

## 2. 本版目标
前端在现有 `Jenkins管理`、`发布模板`、`发布单` 双视图基础上，新增并补齐以下能力：
- 组件管理中新增 `ArgoCD管理`
- 组件管理中新增 `GitOps管理`
- 支持查看、同步、跳转 ArgoCD Application
- 支持查看 GitOps 工作目录、仓库、分支、提交与工作区状态
- 在发布模板中支持“`CD` 留空走 `ArgoCD`，选择绑定则走 `Jenkins`”
- 在新建发布单时，清晰表达 ArgoCD 不直接填写执行器参数，而是复用 `CI` 中映射并勾选的内置字段
- 在发布详情中展示 ArgoCD 的独立执行单元与聚合阶段进度

## 3. 范围
包含：
- `组件管理 -> ArgoCD管理`
- `组件管理 -> GitOps管理`
- `应用管理 -> 管线绑定` 收口为 Jenkins 绑定
- 发布模板页 ArgoCD 文案与校验引导
- 新建发布单页 ArgoCD 参数提示
- 发布详情页 ArgoCD 执行单元状态
- 发布详情页 ArgoCD 管线进度

不包含：
- 前端在线编辑 GitOps 仓库文件
- ArgoCD 事件流 / Pod 日志完整视图
- Helm / values.yaml 通用编辑器
- ArgoCD Application 新建 / 编辑 / 删除

## 5. GitOps 管理页
### 5.1 页面目标
让用户能直接看到平台当前用于 ArgoCD 写回的本地工作目录与 Git 仓库状态，避免“明明配了 GitOps，但不知道平台实际在用哪里”。

### 5.2 展示内容
- GitOps 是否启用
- 工作目录路径
- 当前模式
  - 直接仓库模式
  - 工作根目录模式
- 默认分支
- 提交身份
- 认证账号
- 提交信息模版
- 远端仓库地址
- 当前分支
- 最新提交
- 提交说明
- 工作区是否 dirty
- 未提交变更摘要

### 5.3 操作
- `刷新`
- `编辑提交信息模版`
- 复制工作目录
- 复制远端仓库地址

### 5.4 页面表现
- 若 GitOps 未启用，顶部显示 warning 提示
- 若工作目录不是 Git 仓库，明确展示当前状态，不隐藏问题
- 若工作区 dirty，给出醒目的 warning 提示
- 显示 GitOps 提交信息模版，并说明字段来自标准平台 Key；发布执行时从流程参数中取值
- 有 `component.gitops.manage` 权限时，可直接在页面中编辑模版并保存
- 页面中要明确展示当前目录层级约定：
  - `apps -> 应用目录 -> overlays -> 环境目录`
  - 绑定时选择：`apps/java_nantong`
  - 实际执行路径：`apps/java_nantong/overlays/<env>`

## 7. 发布模板页
### 7.1 文案与引导
当模板启用了 `CD` 且未选择 Jenkins 绑定时：
- 继续显示 `CI 配置` / `CD 配置` 双卡片
- `CD` 卡片文案需明确说明：
  - 平台不会再让用户选择 ArgoCD 的“执行器参数”
  - `CD` 不选择绑定管线时，提示“当前未选择 CD 绑定管线，将按 ArgoCD 执行”
  - `CD` 选择了 Jenkins 绑定时，提示“当前执行器：jenkins”
  - 平台会在执行阶段复用 `CI` 中已映射且勾选的内置字段
  - 平台会自动写回 GitOps 仓库并触发 ArgoCD Sync

### 7.2 模板保存前端反馈
后端返回：
- `cd 选择 argocd 时，ci 绑定管线必须先映射这些内置字段：...`
- `cd 选择 argocd 时，请在 ci 模板参数中勾选这些内置字段：...`

前端应直接显示该错误，不做模糊提示。

## 8. 新建发布单页
### 8.1 基础目标
新建发布单仍然保持：
- 应用
- 发布模板
- 创建者
- 备注

### 8.2 ArgoCD 参数视图规则
当某个执行单元是 `CD + ArgoCD` 时：
- 仍展示 `CD` 卡片
- 卡片标题、绑定信息继续保留
- 提示文案改为：
  - “平台会在执行阶段复用 CI 中映射并勾选的内置字段自动写回 GitOps 仓库”
- 若 `CD` 下没有可填写参数：
  - 不再提示“参数暂不开放”
  - 改为说明“当前执行单元将复用 CI 的内置字段自动执行 GitOps 写回”

### 8.3 交互预期
- 用户看到 `CD=ArgoCD` 时，不会误以为自己还要再填一套 ArgoCD 原生参数
- 用户知道 ArgoCD 所需内置字段来自 `CI` 模板参数，不需要在 `CD` 再填一次

## 9. 发布详情页
### 9.1 执行单元
继续按 `CI` / `CD` 双视图展示：
- 绑定名称
- 执行器
- 状态
- 开始/结束时间

### 9.2 参数快照
- `CI` 参数快照照常展示
- `CD=ArgoCD` 时，如果参数快照中有 `image_version`，也应在 `CD 参数快照` 中看到

### 9.3 执行步骤
保留全局步骤与 `CI/CD` 分组步骤。

### 9.4 管线进度
当 `CD=ArgoCD` 时：
- 不再显示“待接入”
- 直接展示后端回传的聚合阶段：
  - `GitOps 写回`
  - `ArgoCD Sync`
  - `健康检查`
- 这些阶段与 Jenkins Stage 使用同一套表格样式，保持 UI 一致
- `查看日志` 按钮仅对 Jenkins 阶段开放

### 9.5 日志区域
当前规则：
- Jenkins：继续展示流式日志与阶段日志
- ArgoCD：先显示说明型提示

文案建议：
- “当前使用 ArgoCD，当前版本先展示执行进度；事件流/日志视图将在后续版本补齐。”

## 10. 状态展示规范

### 10.2 ArgoCD Health 状态
- `Healthy`：绿色
- `Progressing`：蓝色
- `Degraded`：红色
- `Missing`：橙色
- `Unknown`：灰色

### 10.3 发布详情中的 ArgoCD 聚合阶段
- `GitOps 写回`

- `健康检查`
  - 根据 HealthStatus 映射

## 14. 验收标准
- 左侧菜单可进入 `GitOps管理`
- 可查看 GitOps 工作目录、远端仓库、当前分支、最新提交、工作区状态
- 左侧菜单可进入 `ArgoCD管理`
- 手动同步可正常返回结果并刷新列表
- `应用管理 -> 管线绑定` 中不再允许选择 `argocd` 作为 CD 提供方
- 发布模板中：
  - `CD` 不选绑定管线时，提示将按 ArgoCD 执行
  - `CD` 选了绑定管线时，提示 `当前执行器：jenkins`
- 页面可清楚理解 GitOps 目录层级，并明确环境由标准 Key `env` 传递
- 模板 `CD=ArgoCD` 时，文案能明确说明“复用 CI 中已映射且勾选的内置字段”的逻辑
- 新建发布单页不会错误提示 ArgoCD “参数暂不开放”
- 发布详情页能看到 `CD=ArgoCD` 的聚合阶段进度
- ArgoCD 阶段不展示误导性的 Jenkins 日志按钮

## 15. 设计提醒
- ArgoCD 在前端不应被塑造成“另一套 Jenkins”
- 用户可见的重点应该是：
  - 绑定了哪个 ArgoCD Application
  - 当前 Sync / Health 状态
  - GitOps 是否写回成功
  - CD 执行是否完成
