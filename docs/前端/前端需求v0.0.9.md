# 前端需求文档 v0.0.9（基于 v0.0.8 增量）

## 1. 文档信息
- 版本：`v0.0.9`
- 基线日期：`2026-03-18`
- 上一版本：`/Users/lingyunxieqing/Desktop/gos/docs/前端/前端需求v0.0.8.md`
- 本版主题：`回滚入口 + 发布详情取值进度增强 + ArgoCD/GitOps 可视化收口`

## 2. 本版目标
前端在 `v0.0.8` 的 `ArgoCD管理`、`GitOps管理`、`CI/CD` 双视图发布基础上，继续补齐以下能力：
- `我的应用` 页面支持直接创建回滚发布单
- 回滚操作增加二次确认，避免误触
- 发布单列表、详情页能明确识别“这是一张回滚单”
- 发布单详情新增并强化 `取值进度` 视图
- `取值进度` 不再只展示模板显式勾选字段，而是要覆盖系统内置字段
- `CD=ArgoCD` 时，页面要明确展示内置字段是如何从 `CI` 复用到 `CD`

## 3. 范围
包含：
- `我的应用` 列表页回滚入口
- 发布单列表、发布单详情回滚标识
- 发布单详情 `取值进度`
- `取值进度` 中内置字段自动展示
- `CD=ArgoCD` 的内置字段复用展示

不包含：
- 回滚自动执行
- 回滚专属审批流
- 发布单之间可视化差异对比
- GitOps 事件流时间线

## 4. 我的应用页
文件目标：
- `/Users/lingyunxieqing/Desktop/gos/frontend/src/views/application/ApplicationListView.vue`

### 4.1 新增按钮
每个应用新增：
- `发布`
- `回滚`

### 4.2 回滚按钮规则
- 只有当前用户具备该应用的发布权限时，按钮才可点击
- 未配置可发布模板时，回滚按钮与发布按钮保持同样的禁用规则

### 4.3 二次确认
点击 `回滚` 后弹出确认框：
- 标题：`确认回滚 <应用名称> 吗？`
- 说明：`系统会基于该应用最近一次成功发布的参数，重新创建一张回滚发布单。`
- 按钮：
  - `确认回滚`
  - `取消`

### 4.4 回滚后的跳转
- 回滚建单成功后，前端直接跳转到新建出的回滚发布单详情页
- 成功提示中需要展示来源发布单号

## 6. 发布单详情页
文件目标：
- `/Users/lingyunxieqing/Desktop/gos/frontend/src/views/release/ReleaseOrderDetailView.vue`

### 6.1 基础信息
基础信息区新增：
- `上次发布单号`

### 6.2 发布单号标签
在 `发布单号` 字段内容右侧：
- 若该单为回滚单，则显示红色 `回滚` 标签

### 6.3 取值进度视图
在现有 `参数快照` 下方继续保留：
- `CI 取值进度`
- `CD 取值进度`

### 6.4 取值进度增强规则
`取值进度` 需展示两类字段：
1. 模板中显式勾选并映射的标准 Key
2. 系统内置字段

即使内置字段：
- 没有在模板里手工勾选
- 也没有在页面上作为显式额外参数展示

只要它属于平台启用中的内置字段，就应自动进入 `取值进度`

### 6.5 内置字段展示规则
当前已支持：
- `image_version`
- 后续新增内置字段时，前端不需要额外写死判断，应自动跟随后端返回

### 6.6 `CD=ArgoCD` 时的展示
若当前模板 `CD=ArgoCD`：
- `CD 取值进度` 中要展示内置字段
- 且文案要体现：
  - `沿用 CI`
  - `等待 CI`
  - `CI 失败导致无法复用`

### 6.7 取值进度状态
状态继续使用：
- `pending`
- `running`
- `resolved`
- `failed`
- `skipped`

前端负责做友好中文展示，不显示原始英文枚举。

## 7. 文案要求
### 7.1 回滚相关
- 不使用“重新发布旧单”这类容易误解的文案
- 统一用：
  - `回滚`
  - `回滚发布单`
  - `上次发布单号`

### 7.2 取值进度相关
- 强调“当前值从哪里来”
- 对 `ArgoCD` 场景强调“CD 复用 CI 的内置字段值”
- 避免让用户误以为还要手工填写一遍 `CD` 内置字段

## 8. API 对接

- `POST /applications/{id}/release-orders/rollback`

### 8.2 发布单详情
