## 2. 背景
在生产网络与发布基建隔离的场景下，平台不能默认直连生产环境。

因此前端在 `v0.1` 需要配合后端新增一套“孤岛 Agent”视图，帮助用户理解：
- 哪些生产环境由 Agent 接管
- 哪些发布单正在等待 Agent 领取
- 哪些发布单将执行哪份 Shell 脚本
- Agent 是否在线
- Agent 回传了哪些日志与状态

## 3. 目标
前端需要支持：
1. 小岛/Agent 可视化管理
2. 发布模板选择 `CD provider = island-agent`
3. 发布详情展示 Agent 领取、脚本执行、日志回传过程
4. 对“生产侧未连通”的场景给出明确状态提示
5. 提供平台托管脚本的查看与审计能力
6. 展示脚本变量与渲染结果

## 4. 范围
包含：
- 小岛管理页面
- Agent 管理页面
- 平台脚本管理页面
- 脚本变量定义与版本查看
- 发布模板中 `island-agent` 配置入口
- 发布详情中的 Agent 执行进度与日志

不包含：
- Agent 安装向导
- Agent 自升级页面
- 平台远程交互式 Shell 控制台

### 10.4 空态与失败说明
当没有在线 Agent 时，需要明确提示：
- 当前没有在线 Agent，任务尚未被领取

当 Agent 心跳过期时，需要提示：
- Agent 心跳已过期，请联系运维确认生产节点状态

当脚本下载失败时，需要提示：
- Agent 未能成功拉取平台脚本，请检查网络与脚本版本

当变量渲染失败时，需要提示：
- 脚本变量渲染失败，请检查模板参数与默认值配置

## 12. 交互细节
### 11.1 Agent 状态标签
建议统一状态：
- `在线`
- `离线`
- `心跳过期`
- `执行中`
- `停用`

### 11.2 小岛任务提示
在发布详情页顶部建议增加提示条：
- `该发布单由生产小岛 Agent 主动领取执行`

### 11.3 脚本查看
在模板页、发布详情页都建议支持：
- 查看脚本名称
- 查看脚本版本
- 只读查看脚本正文

目的：
- 让用户知道“待执行的 Shell 脚本”到底是什么
- 降低生产发布的不确定感

### 11.4 变量查看
建议统一支持：
- 查看脚本变量列表
- 查看变量来源
- 查看渲染后的实际值

目的：
- 让用户知道脚本不是黑盒执行
- 明确这次发布到底替换了哪些参数

### 11.5 错误提示
错误提示要尽量说人话，例如：
- `当前生产小岛没有在线 Agent，任务暂未开始`
- `Agent 已领取任务，但长时间未回传结果`
- `Agent 拉取脚本失败，请检查生产网络连通性`
- `脚本变量渲染失败，请检查参数映射与脚本默认值`
- `Agent 执行脚本失败，请查看 CD 日志`

## 15. 结论
`v0.1` 前端需要把“生产孤岛发布”从后台概念，变成用户可理解、可追踪的产品界面。

核心目标不是增加一个新名词，而是让用户在页面上清楚知道：
- 任务是否已经被小岛 Agent 领取
- 这次到底要执行哪份 Shell 脚本
- 这份脚本中的变量最终被替换成了什么
- 生产环境是否在线
- 部署是否正在执行
- 失败到底卡在了哪一步
