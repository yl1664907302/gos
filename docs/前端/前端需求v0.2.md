## 2. 背景
当前平台已经具备：
- Jenkins 触发与跟踪能力
- `CD=ArgoCD` 的 GitOps/Helm 发布能力
- 发布单、回滚、并发执行与预检机制

但生产环境存在特殊约束：
- 发布平台不能直连生产内网
- 发布动作仍然需要落到生产网内部执行
- 现有人工流程依赖“找包、拷包、重启、核对版本”等手工步骤

因此前端在 `v0.2` 需要配合后端新增一套“特殊环境 Agent 执行模式”视图，目标不是替换所有环境的 CD，而是为生产等特殊环境提供一条：
- 平台统一建单与审批
- Agent 主动拉取任务
- 平台统一可视化管理与审计

## 3. 目标
前端需要支持：
1. 特殊环境的执行模式展示与区分
2. Agent 管理页面，展示 Agent 心跳、主机信息、当前任务、最近结果
3. 生产执行任务视图，展示任务排队、领取、执行、结果回传过程
4. 发布模板中新增 `agent_task` 执行模式配置入口
5. 发布详情页展示 Agent 任务进度、脚本信息、变量渲染结果与执行日志
6. 不影响普通环境继续使用 Jenkins / ArgoCD

## 4. 范围
包含：
- Agent 管理页面
- 生产执行任务列表与详情
- 发布模板中 `agent_task` 入口
- 发布详情中的 Agent 任务进度、变量、日志
- 特殊环境执行模式说明与状态标签

不包含：
- Agent 安装向导
- Agent 自升级界面
- 浏览器内交互式终端
- 任意临时命令输入框

## 5. 核心前端概念
### 5.1 执行模式
前端需要支持按环境或模板展示执行模式，例如：
- `gitops`
- `pipeline`
- `agent_task`
- `manual`

说明：
- `agent_task` 仅用于特殊环境
- 普通环境页面不应被这套配置打扰

### 5.2 Agent
Agent 是部署在特殊网络区域中的执行节点。前端只展示其：
- 基础身份
- 在线状态
- 当前任务
- 最近结果
- 运行能力

### 5.3 生产执行任务
“生产执行任务”是平台对特殊环境的一种扩展执行方式。

前端需要把它展示为：
- 由发布单派生出的特殊执行任务
- 由 Agent 主动轮询领取
- 在受控工作目录内执行模板化 Shell 脚本

### 5.4 受控 Shell 任务
前端不得把这套能力展示成“任意远程 Shell”。

页面与文案应统一强调：
- 仅执行平台托管的脚本/命令模板
- 仅允许平台已定义的变量
- 仅在 Agent 配置的工作目录内执行
- 仅由 Agent 主动拉取执行

## 8. Agent 任务配置表单
### 8.1 建议字段
- 执行模式：`agent_task`
- Agent 选择器
- 工作目录
- 制品目录（可选）
- 命令模板 / 脚本模板
- 超时时间
- 失败重试次数
- 变量列表

### 8.2 变量来源
前端需要清楚展示变量来源，例如：
- 发布基础字段
- 标准字库字段
- 固定值
- 内置字段
- 来源成功单（用于回滚）

### 8.3 命令预览
建议支持“渲染预览”，让用户能在模板页预览：
- 原始命令模板
- 变量渲染示例

但不要展示为可随意输入任意 shell。

在列表和详情中建议增加：

- 还是特殊环境的 Agent 执行

## 12. 状态设计
### 12.1 Agent 状态
建议统一状态：
- `在线`
- `离线`
- `执行中`
- `停用`
- `维护中`

### 12.2 任务状态
建议统一状态：
- `待领取`
- `已领取`
- `执行中`
- `执行成功`
- `执行失败`
- `已取消`

### 12.3 空态与异常提示
需要明确提示：
- 当前没有在线 Agent
- Agent 心跳已超时
- 当前没有可领取节点
- 任务执行失败，请查看 Agent 日志
- 当前命令模板不允许在该环境执行

## 13. 权限建议
前端需要配合后端区分：
- 查看 Agent 列表
- 查看 Agent 详情
- 暂停/恢复 Agent
- 查看生产执行任务
- 管理 Agent 任务模板
- 查看 Agent 日志

## 14. 交互细节
### 14.1 非特殊环境不展示 Agent 配置
只有命中 `agent_task` 的场景，才显示 Agent 配置与说明。

### 14.2 页面文案统一
避免出现：
- 任意 Shell
- 远程执行命令
- 手工输入生产命令

统一使用：
- `受控脚本`
- `任务模板`
- `Agent 主动领取`

### 14.3 列表页轻量展示
列表页只展示核心状态，不直接展开渲染命令与变量明细。
细节应进入抽屉或详情页。

## 15. 验收标准
前端侧验收建议：
1. 能识别并展示 `agent_task` 执行模式
2. 能在 Agent 管理页查看在线/离线/执行中状态
3. 能查看 Agent 最近心跳、主机名、IP、当前任务
4. 能在发布详情页查看 Agent 任务进度、日志、变量与脚本信息
5. 普通环境页面不受这套特殊模式干扰
6. 页面文案不把该能力展示成任意远程 Shell
