# 前端需求文档 v0.3（需求设计）

## 1. 文档信息
- 文档版本：`v0.3`
- 基线日期：`2026-03-30`
- 参考版本：`/Users/lingyunxieqing/Desktop/gos/docs/前端/前端需求v0.2.md`
- 本版主题：`发布后 Hook 模块`
- 当前状态：`需求设计，待开发`

## 2. 背景
当前平台已经具备：
- 发布模板、发布单、回滚、重放回滚
- Jenkins / ArgoCD / GitOps 发布链路
- Agent 脚本管理、任务管理、常驻任务与历史任务

但发布完成后仍然缺少一层“发布后动作”能力，例如：
- 发布成功后执行烟雾测试
- 发布成功后清理缓存、刷新配置、触发二次同步
- 发布失败后执行补偿动作
- 针对特殊环境执行发布后脚本或管线

因此 `v0.3` 需要在**发布模板**中新增一个“Hook 模块”，用于在主发布流程结束后执行扩展动作。

## 3. 目标
前端需要支持：
1. 在发布模板中配置发布后 Hook
2. 在发布单创建和详情中展示 Hook 预期信息与实际执行结果
3. 在发布单详情中展示 Hook 执行进度、日志、失败原因
4. 区分 Hook 的触发条件、失败策略、执行顺序
5. 支持 Hook 复用现有脚本管理、Agent 任务与 Jenkins 管线

## 4. 范围
包含：
- 发布模板中的 Hook 配置区
- 发布单详情中的 Hook 进度区
- 发布单列表中的 Hook 结果摘要
- Agent / Jenkins 作为 Hook 执行目标的配置选择
- Hook 日志与变量预览展示

不包含：
- Hook 可视化编排画布
- Hook DAG / 并行图形设计器
- 浏览器直接输入任意远程命令
- 多阶段 Hook 编排（本版先聚焦发布后阶段）

## 5. 核心概念
### 5.1 Hook 阶段
本版只支持一个阶段：
- `post_release`

含义：
- 主发布流程执行结束后才开始执行 Hook
- 主发布流程包括：CI / CD / 健康检查 / Agent 发布任务

### 5.2 Hook 触发条件
模板中每个 Hook 需要配置触发条件：
- `on_success`：仅主发布成功后执行
- `on_failed`：仅主发布失败后执行
- `always`：无论主发布成功失败都执行

### 5.3 Hook 执行类型
首版建议支持：
- `jenkins_pipeline`：触发 Jenkins 管线
- `agent_task`：创建 Agent 任务执行脚本

说明：
- `agent_task` 优先复用现有脚本管理与 Agent 体系
- 暂不支持任意 Shell 文本直接输入

### 5.4 Hook 失败策略
模板中每个 Hook 需要配置失败策略：
- `block_release`：Hook 失败则整张发布单最终失败
- `warn_only`：Hook 失败仅告警，发布单保持主流程结果

### 5.5 Hook 执行顺序
- 模板中支持配置多个 Hook
- 首版按 `sort_no` 串行执行
- 前端需要展示明确顺序，例如：`Hook 1 / Hook 2 / Hook 3`

## 8. 发布单详情页
目标页面：
- `/Users/lingyunxieqing/Desktop/gos/frontend/src/views/release/ReleaseOrderDetailView.vue`

### 8.1 新增 Hook 进度区
在详情页增加：
- `Hook 执行进度`

要求：
- 作为发布单详情页中的独立卡片展示
- 位于主发布进度之后，能够一眼看到 Hook 是否已开始、是否仍在执行、是否影响最终结果
- 若存在多个 Hook，需要按顺序展示执行链路，而不是只给最终摘要

状态建议：
- `待执行`
- `已跳过`
- `执行中`
- `成功`
- `失败`
- `失败但已忽略`

### 8.2 Hook 列表展示
每个 Hook 展示：
- 顺序号
- Hook 名称
- Hook 类型
- 触发条件
- 失败策略
- 当前状态
- 开始时间
- 结束时间
- 耗时
- 执行摘要

### 8.3 Hook 日志
详情页支持展开日志：
- Jenkins Hook：展示触发信息、构建号、返回摘要
- Agent Hook：展示 stdout/stderr、退出码、目标 Agent

### 8.4 Hook 变量展示
每个 Hook 可展开查看：
- 变量 Key
- 来源类型
- 解析后的值

### 8.5 Hook 进度 UI 线框
建议在发布单详情页中采用与现有主进度卡一致的轻量白色卡片，结构如下：

```text
Hook 执行进度

整体状态：执行中 / 成功 / 失败 / 部分失败
摘要：共 3 个 Hook，1 个执行中，1 个成功，1 个待执行

1. 发布后冒烟检查        Jenkins 管线     仅成功后触发   执行中
   开始时间：2026-03-31 10:21:33
   失败策略：失败阻断
   执行摘要：已触发 Jenkins #182，等待结果
   [查看变量] [查看日志]

2. 刷新边缘缓存          Agent 任务       总是触发       待执行
   失败策略：失败告警
   执行摘要：等待前一个 Hook 完成
   [查看变量]

3. 发布结果通知          Agent 任务       仅失败后触发   已跳过
   执行摘要：主发布成功，未命中触发条件
```

展示要求：
- 顶部先展示 Hook 总体摘要，不要只展示列表
- 每个 Hook 单独一行卡片，按 `sort_no` 从上到下排列
- `执行中`、`失败`、`失败但已忽略` 必须有明显状态色
- `查看变量` 与 `查看日志` 用折叠面板或抽屉展开，不要跳页面
- 若 Hook 数量较多，默认展开当前执行中和失败项，其余可折叠

### 8.6 Hook 进度卡字段
详情页 Hook 卡建议最少展示：
- Hook 名称

- Hook 类型

## 11. 表单与样式要求
- Hook 配置区沿用发布模板页面现有表单语言
- 减少额外灰底嵌套块，保持白色表单面板风格
- 多个 Hook 使用轻量卡片，不要做厚重的驾驶舱式装饰
- 日志区仍可保留深色代码块
- Hook 执行进度卡参考发布单详情页现有状态卡风格

## 12. 权限建议
建议新增前端权限点：
- `release.template.hook.view`
- `release.template.hook.edit`
- `release.hook.log.view`

## 13. 首版限制
首版建议明确限制：
1. 只支持 `post_release`
2. 只支持串行 Hook
3. 只支持 `jenkins_pipeline / agent_task`
4. Hook 本体只能在模板中配置
5. 建单时不允许临时新增/删除 Hook

## 14. 验收要点
1. 发布模板中能新增、编辑、排序多个 Hook
2. 发布单创建页能预览模板中 Hook 摘要
3. 发布单详情能展示 Hook 执行进度与日志
4. Hook 失败策略能在前端清晰区分
5. Agent Hook 能和现有脚本管理、任务管理串起来
6. 发布单详情页必须能直接看到 Hook 执行进度，不需要跳转二级页面
7. 页面风格与现有发布单页面保持统一
