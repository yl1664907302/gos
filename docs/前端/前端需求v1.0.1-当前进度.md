## 3. 当前前端整体状态
当前前端已从“基础发布平台原型”升级为“可真实承载发布流程”的控制台，核心能力已经覆盖：
- 官网与产品化入口
- 应用 / 项目管理
- 发布模板配置
- 发布单创建 / 列表 / 详情
- 审批工作台
- Agent 管理 / 任务 / 脚本
- 通知源 / Markdown 模板 / 通知 Hook
- GitOps / Jenkins / ArgoCD 相关配置页
- 基于系统设置发布环境的环境级权限与筛选

整体上，前端已不再是单纯的 CRUD 页面集合，而是具备：
- 发布视角
- 审批视角
- Agent 运维视角

- 官网展示视角

## 6. 当前仍需继续优化的前端体验点
虽然核心功能已基本完成，但当前仍有明确的体验优化空间：

### 6.1 发布前预演
当前缺口：
- 用户仍需要“创建后再看详情”才能完整判断这次发布会怎么执行

建议补齐：
- 创建页增加 `发布前预演` 卡片
- 提前展示：
  - 是否需要审批
  - 是否会排队
  - 最终参数取值
  - 执行单元摘要
  - Hook 执行预期
  - GitOps 目标摘要

### 6.2 失败后续跑能力
当前缺口：
- 失败单更多还是依赖重新建单或回滚

建议补齐：
- `从失败步骤继续`
- `按成功快照续跑`

### 6.3 常用发布预设
当前缺口：
- 用户重复填同一套参数成本偏高

建议补齐：
- 保存“我的发布预设”
- 一键带出模板、环境、参数、分支

### 6.4 详情页时间线表达
当前缺口：
- 细节已经很多，但“从开始到结束”的线性叙事感还可以再加强

建议补齐：
- 统一时间线视图：
  - 建单
  - 审批
  - 排队
  - CI
  - CD
  - Hook
  - 完成/失败/取消

### 6.5 按钮禁用原因统一提示
当前缺口：
- 部分按钮虽然已禁用，但用户不能一眼知道原因

建议补齐：
- 统一 hover / tooltip 原因说明：
  - 无权限
  - 模板有审批
  - 排队中
  - 环境未授权
  - 当前状态不可操作
