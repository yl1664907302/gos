## 2. 背景
当前创建发布单流程已经支持：
- 选择应用、模板、环境
- 动态渲染 CI / CD 参数
- 查看模板摘要
- 使用极速发布

但用户仍然需要在“创建发布单后”进入详情页，才能完整知道：
- 本次是否需要审批
- 是否会排队
- 最终参数会怎么取值
- 会执行哪些 Hook
- GitOps 会改哪个分支、哪个路径

这导致：
- 建单前认知成本高
- 容易因为参数理解偏差产生误操作
- 极速发布缺少“事前确认”的透明度

因此需要在创建页引入 `发布前预演`。

## 3. 目标
前端需要支持：
1. 在创建发布单页实时展示本次发布预演结果
2. 在用户填完关键字段后自动刷新预演
3. 在视觉上明确区分：
   - 可直接发布
   - 需要审批
   - 将进入排队
   - 存在阻断项
4. 让 `创建发布单` 与 `极速发布` 都建立在同一份预演结果基础上

## 4. 范围
包含：
- 创建页新增 `发布前预演` 卡片
- 自动预演与手动刷新交互
- 参数最终取值展示
- 执行单元摘要展示
- Hook 执行预期展示
- 审批 / 排队 / 阻断项展示
- GitOps 目标摘要展示

不包含：
- 真实执行前的二次确认弹窗重构
- 预演结果持久化存档
- 可视化 diff 编辑器
- 创建页直接修改模板结构

## 6. 交互设计

### 6.1 触发时机
当以下字段具备后自动触发预演：
- `application_id`
- `template_id`
- `env_code`

如果模板还存在必填动态参数：
- 初次也先发起一次预演
- 缺失值在结果中标记为“待填写”
- 参数变化后做 `300ms ~ 500ms` 防抖刷新

### 6.2 手动刷新
预演卡片需要提供：
- `刷新预演` 按钮

适用场景：
- 用户修改了多个参数后希望手动确认
- 网络抖动导致自动预演失败

### 6.3 创建按钮联动
- `创建发布单`：允许在预演存在 `warn` 的情况下继续
- `极速发布`：
  - 模板存在审批人时仍禁用
  - 若预演存在阻断项，按钮不可执行

## 7. 预演卡片结构
建议放置位置：
- 模板摘要卡下方
- CI / CD 参数卡上方

建议结构如下：

### 7.1 顶部总览
展示：
- 总体状态：
  - `可直接发布`
  - `需审批后发布`
  - `将进入排队`
  - `存在阻断项`
- 一行摘要：
  - `2 个执行单元 · 3 个 Hook · 1 个风险项`

### 7.2 关键信息卡
至少包含：
- 审批
- 并发 / 队列
- 执行单元
- Hook
- GitOps

### 7.3 可展开详情
包含：
- 预审项列表
- 最终参数值列表
- Hook 预期执行列表
- GitOps 目标信息

## 8. 展示字段设计

### 8.1 审批摘要
展示：
- 是否启用审批
- 审批模式
- 审批人名称
- 当前用户是否可走极速发布

### 8.2 并发 / 队列摘要
展示：
- 是否启用并发控制
- 锁范围
- 锁 key
- 是否会进入排队
- 前面还有几单
- 冲突单号

### 8.3 执行单元摘要
展示：
- 将生成几个执行单元
- CI 绑定名称
- CD 绑定名称
- 绑定是否为模板快照回退

### 8.4 参数摘要
展示：
- 参数名
- 参数最终值
- 来源：
  - 用户填写
  - 固定值
  - 平台内置
  - 快照回填
- 当前状态：
  - 已解析
  - 待填写
  - 冲突

### 8.5 Hook 摘要
展示：
- Hook 名称
- 类型
- 是否命中当前环境
- 是否会执行
- 失败策略
- 跳过原因

### 8.6 GitOps 摘要
展示：
- 目标分支
- Values / YAML 路径
- 主要替换字段
- image_version 来源

## 9. 状态表达建议
预演项状态统一使用：
- `pass`
- `warn`
- `blocked`

前端文案建议：
- `通过`
- `提醒`
- `阻断`

说明：
- `warn` 不阻止建单，但需要明显提醒
- `blocked` 明确阻止极速发布，必要时也阻止建单

## 10. 视觉建议
- 总体使用轻量卡片，不要弹窗
- 顶部总览区优先级最高
- `blocked` 要有明显但克制的红色提醒
- `warn` 使用暖色提示，不要和失败态混淆
- Hook / GitOps 摘要可采用小型信息卡并排展示

## 11. 与现有页面的关系
本次不替代：
- 发布单详情页中的完整日志和执行过程

本次新增：
- 建单前“计划层”可视化

也就是说：
- 创建页负责回答 `将会怎么发`
- 详情页负责回答 `已经怎么发`

## 12. 验收标准
以下视为前端完成：
1. 用户在填写关键字段后能自动看到预演结果
2. 用户能一眼看出本次是否需要审批 / 排队 / 有阻断项
3. 参数、Hook、GitOps 预期信息能在创建前看到
4. 创建页与极速发布都能复用同一份预演结果
5. 预演失败不会导致整个创建页不可用，可手动刷新重试
