## 2. 背景
当前平台已支持：
- 发布单创建与执行
- 极速发布
- 执行前预审
- 发布模板、Hook、审批、并发队列

但在真实企业发布中，部分发布会伴随数据库变更语句。当前平台缺少一层轻量、标准化的 SQL 留档与执行前审核能力，导致：
- 发布单中是否涉及 SQL 无法沉淀
- 执行前无法统一做 SQL 风险检查
- 详情页与审计链路无法明确记录 SQL 变更责任

本期目标不是执行 SQL，而是先完成：
- SQL 备案
- SQL 审计
- SQL 语法与基础安全检查
- 执行前阻断能力

## 3. 目标
前端需要支持：
1. 在普通创建发布单时可选填写 SQL 备案
2. SQL 默认关闭，默认口径为“无 SQL 变更”
3. 极速发布路径默认且强制为“无 SQL 变更”
4. 在发布单详情页执行前，预审第一步展示 SQL 审核进度
5. SQL 审核不通过时，明确阻断执行并展示原因
6. 在详情页、列表页中展示发布单是否包含 SQL 备案及审核状态

## 4. 范围
包含：
- 创建页 SQL 备案卡片
- 极速发布与 SQL 备案的互斥规则
- 详情页执行前预审中的 SQL 审核展示
- SQL 审核结果展示
- 列表页 SQL 标识摘要

不包含：
- SQL 真正执行
- 数据库实例连接与在线 DDL
- SQL 回滚自动执行
- SQL 文件上传器（本期可先仅支持文本）

## 5. 产品规则

### 5.1 普通创建发布单
- 默认：`无 SQL 变更`
- 用户可主动切换为：`包含 SQL 备案`
- 若用户未开启 SQL 备案，则该发布单后续预审中 SQL 审核直接通过并显示“无 SQL 备案”

### 5.2 极速发布
- 极速发布默认且强制为：`无 SQL 变更`
- 极速发布路径不允许携带 SQL 备案内容
- 若用户已经填写 SQL 备案，则：
  - `极速发布` 按钮禁用
  - 提示：`当前发布单已配置 SQL 备案，请使用标准创建发布单`

### 5.3 执行前审核
- 点击详情页 `执行` 按钮后，先进入预审流程
- SQL 审核作为预审第一项
- SQL 审核不通过时：
  - 阻断执行
  - 弹出或内嵌显示审核失败原因
- SQL 审核通过后，继续执行后续预审项

### 6.3 展开后字段
建议字段：
- 标题
- 变更类型
  - `DDL`
  - `DML`
  - `查询校验`
  - `初始化数据`
- SQL 内容
- 变更说明
- 回滚 / 补救说明

### 6.4 校验反馈
创建页填写 SQL 时可做轻量即时校验：
- 空文本提醒
- 基本语句识别
- 仅前端提示，不作为最终执行判断依据

说明：
- 最终以执行前预审中的后端 SQL 审核结果为准

## 7. 创建页与极速发布联动
### 7.1 普通创建按钮
- 普通创建允许带 SQL 备案创建
- 创建时仅保存备案，不触发执行

### 7.2 极速发布按钮
当以下任一情况成立时，`极速发布` 不可用：
- 模板存在审批人
- 当前已配置 SQL 备案
- 当前预演或必填项未通过

建议禁用提示：
- `当前发布单已配置 SQL 备案，暂不支持极速发布`

### 9.2 执行前预审弹层 / 区块
点击 `执行` 后，预审中新增第一项：
- `SQL 审核`

状态：
- `审核中`
- `通过`
- `警告`
- `阻断`

文案示例：
- `正在进行 SQL 审核...`
- `SQL 审核通过，本次发布未发现阻断风险`
- `SQL 审核未通过：DELETE 语句缺少 WHERE 条件`

### 9.3 审核失败后的行为
- 不允许继续执行
- 保留当前发布单状态不变
- 用户可返回查看并修改 SQL 备案（后续版本可支持编辑）

## 10. 审核结果展示规范
### 10.1 状态分级
统一使用：
- `pass`
- `warn`
- `blocked`

前端文案：
- `通过`
- `警告`
- `阻断`

### 10.2 结果信息
每条 SQL 审核项至少展示：
- 规则名称
- 状态
- 说明文案
- 对应语句片段（可选）

## 11. 动效与体验要求
- 点击执行后，预审项要逐项展示，不要瞬间全部跳出
- SQL 审核作为第一项，建议带加载动画
- 阻断项要明显，但不使用过强恐吓色
- 审核通过时要让用户明确知道“不是跳过了，而是已经审完了”

## 12. 验收标准
以下视为前端完成：
1. 创建页支持 SQL 备案，默认无 SQL
2. 极速发布与 SQL 备案互斥
3. 列表页与详情页可识别发布单是否带 SQL 备案
4. 执行前预审第一项为 SQL 审核
5. SQL 审核失败时阻断执行并展示原因
6. 没有 SQL 的发布单能在预审中明确显示“无 SQL 备案”
