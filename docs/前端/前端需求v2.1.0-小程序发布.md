# 前端需求文档 v2.1.0（小程序发布）

## 1. 文档信息
- 文档版本：`v2.1.0`
- 基线日期：`2026-04-29`
- 参考版本：`/Users/lingyunxieqing/Desktop/gos/docs/后端/后端需求v2.1.0-小程序发布.md`
- 本版主题：`多平台小程序发布管理视图`
- 当前状态：`规划中`

## 2. 背景

后端设计上，小程序发布作为 Release Order 的一个新 OperationType（`miniprogram_publish`），复用现有的审批、状态流转、流水线阶段。前端需要在以下几个触点做好适配和新增。

## 3. 目标

1. 提供小程序配置管理页面，让用户配置 AppID、密钥等平台凭证
2. 发布单创建流程支持选择「小程序发布」类型
3. 发布单详情展示小程序特有状态（上传、审核中、审核驳回、发布中）
4. 提供版本历史页面，查看每个版本的上传/审核/发布记录
5. 审核结果的明确可视化（通过/驳回 + 驳回原因）
6. 全量发布 / 分阶段发布的操作入口

## 4. 范围

包含：
- 小程序配置管理页（Application 子页面或独立管理页）
- 发布单创建页适配（新增小程序发布类型选择）
- 发布单详情页适配（小程序状态时间线、操作按钮）
- 版本历史查看
- 侧边栏导航新增小程序入口

不包含：
- 小程序开发者工具的 GUI 集成
- 小程序体验版扫码入口（初版可为链接）
- 实时构建日志展示（复用现有 Jenkins 日志能力）

## 5. 页面设计

GPS：`应用管理 → 我的应用 → 某应用 → 小程序发布配置`

**页头**
- 标题：`小程序发布配置`
- 右侧操作：`新建配置`（当前无配置时）/ `编辑配置`（有配置时）

**无配置状态**

```
┌─────────────────────────────────────┐
│                                     │
│          [小程序图标占位]             │
│                                     │
│       当前应用尚未配置小程序发布       │
│       配置后可创建小程序发布工单       │
│                                     │
│          [新建配置]                  │
│                                     │
└─────────────────────────────────────┘
```

**有配置状态 — 信息卡片**

```
┌─────────────────────────────────────┐
│  小程序配置                  [编辑]  │
│  ──────────────────────────────── │
│  平台        微信小程序             │
│  小程序名称   某某商城               │
│  AppID       wx1234567890          │
│  项目路径     /miniprogram          │
│  状态        启用 ●                │

└─────────────────────────────────────┘
```

**新建/编辑弹窗**

透明玻璃 Modal，自定义标题栏：

```
┌ 新建小程序配置 ─────────────── [×] ─┐
│                                     │
│  平台类型    [微信小程序 ▼]          │
│  小程序名称  [________________]      │
│  AppID      [________________]      │
│  AppSecret  [________________] 👁    │
│  上传密钥    [选择文件] 未选择文件    │
│             或 [粘贴密钥内容]         │
│  项目路径    [________________]      │
│              Git仓库中小程序项目相对   │
│              路径，如 miniprogram/    │
│                                     │
│  状态        [启用 ▼]               │
│                                     │
│              [取消]  [保存]          │
└─────────────────────────────────────┘
```

说明：
- `AppSecret` 和 `上传密钥` 为敏感字段，输入框默认 mask，带眼睛图标切换显示
- 平台类型下拉：微信小程序、支付宝小程序（灰）—— 未实现平台置灰标识
- 遵循 CLAUDE.md 的 Modal 规范：无默认 Ant 底栏，右侧操作按钮在标题栏
- 表单不用嵌套 card，直接平铺

**删除配置**

点击删除需二次确认，提示「删除后该应用将无法创建小程序发布工单」。

### 5.2 发布单创建适配

路由：`/releases/new`
涉及文件：`frontend/src/views/release/ReleaseOrderCreateView.vue`

**改动点**

发布类型下拉新增「小程序发布」选项：

```
发布类型：[向后端服务部署 ▼]
         ├── 向后端服务部署
         ├── 回滚
         ├── 重放
         └── 小程序发布    ← 新增
```

**选择「小程序发布」后的表单差异**

原有表单字段（Git Ref、环境、镜像Tag 等）替换为：

```
发布类型：小程序发布
所属应用：[选择应用 ▼]          ← 只列出 ArtifactType=miniprogram 且有配置的应用
小程序平台：微信小程序            ← 根据应用配置自动回填，不可编辑
AppID：wx1234567890              ← 只读回填
版本号：[1.2.3________________]  ← 手动输入或由 CI 构建参数自动填充
版本描述：[___________________]  ← 选填
发布模板：[选择模板 ▼]           ← 小程序发布模板（如果有）
审批模式：[任意一人审批 ▼]
审批人：[选择用户 ▼]
```

```
│  │                                     │

│     审核通过后执行发布                   │
│                                        │

```

**审核驳回状态**

```
┌─ 发布进度 ─────────────────────────────┐
│                                        │
│  ● ...                                 │
│  ✕ 审核驳回        2026-04-29 12:30    │
│    驳回原因：涉及虚拟支付，需补充资质     │
│    [修改后重新提交]                      │
│                                        │
└────────────────────────────────────────┘
```

**操作按钮（根据当前状态展示）**

| 状态 | 可用操作 |
|---|---|
| 草稿 | 编辑 / 提交审批 |
| 已审批 / CI 构建成功 | 提交审核（人工触发，或自动） |
| 审核通过 | 全量发布 / 分阶段发布 |
| 已发布 | 版本回滚 |
| 审核驳回 | 修改后重新提交 |

按钮遵循透明玻璃风格（`claude.md` 中 Modal 按钮规范）。

### 5.4 版本历史页

路由：`/miniprogram/versions?app_id=xxx`
GPS：`组件管理 → 小程序发布 → 版本历史`

**页面头部**
- 标题：`版本历史`
- 右上角：应用选择下拉 + 平台筛选

**表格**

| 版本号 | 所属应用 | 平台 | 发布工单 | 状态 | 提交时间 | 操作 |
|---|---|---|---|---|---|---|
| 1.2.3 | 某某商城 | 微信 | #042 | 已发布 | 04-28 14:00 | 详情 |
| 1.2.2 | 某某商城 | 微信 | #038 | 审核驳回 | 04-25 10:00 | 详情 |
| 1.2.1 | 某某商城 | 微信 | #035 | 已发布 | 04-20 09:00 | 详情 |

**状态标签**

| 状态 | 标签颜色 | 标签文案 |
|---|---|---|
| `built` | 灰色 | 已构建 |
| `uploaded` | 蓝色 | 已上传 |
| `reviewing` | 橙色 | 审核中 |
| `approved` | 绿色 | 审核通过 |
| `rejected` | 红色 | 审核驳回 |
| `published` | 深绿 | 已发布 |

遵循 CLAUDE.md 的表格规范：方角、深色表头、固定操作列不透底。

## 9. UI 规范遵循

以下规范必须严格遵守（参考 CLAUDE.md）：

- 页头一行：标题左，操作右
- 搜索/查询用透明玻璃风格，不恢复旧式卡片 + 行内表单
- 表格方角，全局 `style.css` 已有重置，页面级 scoped 样式不引入 `border-radius`
- Modal 无默认 Ant 底栏，自定义标题栏按钮，透明玻璃按钮
- 避免多余嵌套卡片、避免重复模块标题如 `01 · xxx`
- 按钮文字紧凑，状态标签简洁
