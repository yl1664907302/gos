# Vue3 + Ant Design Vue 前端需求文档 v0.0.2

## 1. 文档信息
- 版本: `v0.0.2`
- 对应后端基线日期: `2026-03-12`
- 上一版本: `/Users/lingyunxieqing/Desktop/gos/docs/前端需求v0.0.1.md`
- 本版主题: 在“应用管理 > 我的应用”下新增“管线绑定”模块，完成绑定关系 CRUD

## 2. 版本目标
在保留 v0.0.1 应用管理能力不变的基础上，新增应用绑定管线管理：
- 支持查看某应用下 CI/CD 绑定列表
- 支持新增绑定
- 支持编辑绑定
- 支持删除绑定
- 支持按类型、提供方、状态筛选与分页
- 新增 `组件管理 > Jenkins管理` 菜单与页面，展示后端定时同步的 Jenkins 管线数据

## 3. 范围与边界
包含功能:
- 绑定列表查询（分页）
- 绑定创建
- 绑定详情查看
- 绑定更新
- 绑定删除
- Jenkins 管线下拉选择（用于 Jenkins 类型绑定）
- Jenkins 管线管理只读展示（数据来自后端定时同步）

不包含功能:
- 触发 Jenkins/ArgoCD 执行
- ArgoCD 实时可用性校验
- 发布流程编排界面

## 4. 菜单与路由增量
- 一级菜单: `应用管理`
- 二级菜单: `我的应用`（保持不变）
- 一级菜单: `组件管理`
- 二级菜单: `Jenkins管理`
- 新增页面路由:
- `/applications/:id/pipeline-bindings`: 管线绑定列表页（当前应用上下文）
- `/components/jenkins`: Jenkins 管线管理页（只读）

页面入口建议:
- 在“我的应用”列表操作列新增 `管线绑定` 按钮，跳转到 `/applications/:id/pipeline-bindings`
- 在“应用详情”页新增 `管线绑定` 按钮，跳转到同一路由
- 在左侧菜单新增 `组件管理 > Jenkins管理`，点击进入 `/components/jenkins`

## 5. 前端业务对象

### 5.1 PipelineBinding
- `id: string`
- `name: string`（管线名称，前端展示主字段）
- `application_id: string`
- `application_name: string`（由后端返回）
- `binding_type: string`，枚举 `ci | cd`
- `provider: string`，枚举 `jenkins | argocd`
- `pipeline_id: string`
- `external_ref: string`
- `trigger_mode: string`，枚举 `manual | webhook`
- `status: string`，枚举 `active | inactive`
- `created_at: string`
- `updated_at: string`

### 5.2 Pipeline（用于 Jenkins 下拉）
- `id: string`
- `job_name: string`
- `job_full_name: string`
- `status: string`

## 6. 功能需求

### 6.1 管线绑定列表页
功能:
- 顶部展示当前应用信息（至少展示 `application_name`、`application_id`）
- 筛选条件:
- `binding_type`（下拉：全部/ci/cd）
- `provider`（下拉：全部/jenkins/argocd）
- `status`（下拉：全部/active/inactive）
- 表格字段:
- `name`
- `application_name`
- `binding_type`
- `provider`
- `pipeline_id`
- `external_ref`
- `trigger_mode`
- `status`
- `updated_at`
- 操作列:
- `查看`
- `编辑`
- `删除`
- 顶部操作:
- `新增绑定`

分页:
- `page` 默认 1
- `page_size` 默认 20
- `page_size` 最大 100

### 6.2 新增绑定
展示方式建议:
- 使用 `a-modal` 表单

表单字段:
- `binding_type`（必填，`ci/cd`）
- `name`（选填）
- `provider`（条件必填）
- `pipeline_id`（条件必填）
- `external_ref`（条件必填）
- `trigger_mode`（必填，`manual/webhook`）
- `status`（必填，默认 `active`）

动态规则:
- 当 `binding_type=ci`:
- `provider` 固定 `jenkins`（不可编辑）
- 必填 `pipeline_id`
- 不展示/禁用 `external_ref`
- 当 `binding_type=cd`:
- `provider` 可选 `jenkins/argocd`（默认 `argocd`）
- `provider=jenkins` 时必填 `pipeline_id`，禁用 `external_ref`
- `provider=argocd` 时必填 `external_ref`，禁用 `pipeline_id`

提交:
- 调用 `POST /applications/{id}/pipeline-bindings`
- 成功后关闭弹窗、提示成功、刷新列表

### 6.3 查看绑定详情
- 调用 `GET /pipeline-bindings/{binding_id}`
- 可使用详情抽屉或只读弹窗展示完整字段

### 6.4 编辑绑定
- 打开编辑弹窗前先调用 `GET /pipeline-bindings/{binding_id}` 回填
- 可编辑字段:
- `name`
- `provider`（仅 `cd` 有效，`ci` 固定 jenkins）
- `pipeline_id`
- `external_ref`
- `trigger_mode`
- `status`
- 提交调用 `PUT /pipeline-bindings/{binding_id}`

### 6.5 删除绑定
- 点击删除后进行二次确认
- 调用 `DELETE /pipeline-bindings/{binding_id}`
- 成功后刷新当前列表页

### 6.6 Jenkins 管理页（只读）
功能:
- 展示后端定时从 Jenkins 同步到数据库的管线列表
- 筛选条件:
- `name`（按管线名模糊搜索）
- `status`（下拉：全部/active/inactive）
- 表格字段:
- `job_name`
- `job_full_name`
- `status`
- `last_synced_at`
- `last_verified_at`
- `updated_at`
- 远程分页:
- `page` 默认 1
- `page_size` 默认 20，最大 100

交互要求:
- 页面初始化自动拉取列表
- 支持查询、重置、分页联动
- 该页面不提供“新增/编辑/删除管线”操作
- 可选提供“刷新”按钮（仅重新查询后端，不直接操作 Jenkins）

## 7. API 对接要求

### 7.1 查询绑定列表
- 方法: `GET /applications/{id}/pipeline-bindings`
- Query:
- `binding_type?: ci|cd`
- `provider?: jenkins|argocd`
- `status?: active|inactive`
- `page?: number`
- `page_size?: number`
- 响应:
- `data: PipelineBinding[]`
- `page`
- `page_size`
- `total`

### 7.2 创建绑定
- 方法: `POST /applications/{id}/pipeline-bindings`
- Body 字段:
- `binding_type`
- `name?`
- `provider?`
- `pipeline_id?`
- `external_ref?`
- `trigger_mode`
- `status`
- 响应: `{ data: PipelineBinding }`

说明:
- `application_name` 不由前端传，后端自动写入并在响应返回。

### 7.3 查询绑定详情
- 方法: `GET /pipeline-bindings/{binding_id}`
- 响应: `{ data: PipelineBinding }`

### 7.4 更新绑定
- 方法: `PUT /pipeline-bindings/{binding_id}`
- Body 字段:
- `name?`
- `provider?`
- `pipeline_id?`
- `external_ref?`
- `trigger_mode?`
- `status?`
- 响应: `{ data: PipelineBinding }`

### 7.5 删除绑定
- 方法: `DELETE /pipeline-bindings/{binding_id}`
- 响应: `204 No Content`

### 7.6 Jenkins 管线下拉数据
- 方法: `GET /pipelines`
- 建议查询参数:
- `provider=jenkins`
- `status=active`
- `page=1`
- `page_size=100`
- 前端展示文案: `job_name (job_full_name)`
- 选中值: `pipeline.id`

### 7.7 Jenkins 管理页列表数据
- 方法: `GET /pipelines`
- Query:
- `provider=jenkins`（固定）
- `name?: string`
- `status?: active|inactive`
- `page?: number`
- `page_size?: number`
- 响应:
- `data: Pipeline[]`
- `page`
- `page_size`
- `total`

## 8. 错误处理与提示文案
- `400`: 表单校验失败或组合非法，展示后端 `error`
- `404`: 应用/绑定/管线不存在，提示后刷新或返回上页
- `409`: 唯一冲突（如同一应用重复绑定同一类型），提示“该类型绑定已存在”
- `500`: 提示“系统异常，请稍后重试”

## 9. UI 约束（继承 v0.0.1）
- 保持左侧菜单布局与企业后台风格不变
- 使用 Ant Design Vue 组件:
- 列表: `a-table` + `a-pagination`
- 查询: `a-form` + `a-select`
- 新增/编辑: `a-modal` + `a-form` + `a-select` + `a-input`
- 确认删除: `a-popconfirm`
- 反馈: `message`
- 禁止内联 style，沿用现有 token、间距与圆角规范
- 查询结果表格字段支持拖拽调节列宽（表头右侧拖拽）

- 优先实现“列表 + 新增 + 删除”主流程，再补“详情 + 编辑”
