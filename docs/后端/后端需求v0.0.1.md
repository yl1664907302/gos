# 后端需求 v0.0.1（基于当前进度）

## 1. 文档信息
- 文档版本：`v0.0.1`
- 基线日期：`2026-03-12`
- 项目：`gos`（内部轻量部署平台后端）
- 范围：仅 `application` 模块

## 2. 目标
在 DDD/Clean Architecture 约束下，交付可运行的应用管理后端，支持应用的增删改查、分页查询、Swagger 文档、MySQL/SQLite 存储。

## 3. 技术与架构约束
- 语言：`Golang`
- Web：`Gin`
- 存储访问：`database/sql`
- 数据库：`MySQL`（默认）与 `SQLite`
- 文档：`swaggo/swag` + `gin-swagger`
- 分层：`domain / usecase / infrastructure / interfaces / bootstrap`
- 原则：优先标准库、可读性优先、避免过度抽象

## 4. 当前范围（v0.0.1）
已包含：
- 应用实体与状态约束
- 应用 CRUD 接口
- 应用分页查询接口
- MySQL/SQLite 双存储支持
- MySQL 启动连通性重试检查
- 健康检查接口
- Swagger 路由与文档生成

不包含：
- 发布（release）流程
- 部署（deploy）流程
- 运行时（runtime）管理
- 环境（environment）管理
- 鉴权与权限模型

## 5. 领域模型需求
`Application` 字段：
- `id: string`
- `name: string`
- `key: string`（唯一）
- `repo_url: string`
- `description: string`
- `owner: string`
- `status: active | inactive`
- `artifact_type: string`
- `language: string`
- `created_at: time`
- `updated_at: time`

核心规则：
- 创建时 `name`、`key` 必填
- `status` 仅允许 `active/inactive`
- 创建时未传 `status`，默认 `active`
- `key` 唯一冲突返回 409

## 6. HTTP API 需求
### 6.1 系统接口
| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/healthz` | 健康检查，返回 `{"status":"ok"}` |
| GET | `/swagger/*any` | Swagger UI |

### 6.2 应用接口
| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/applications` | 创建应用 |
| GET | `/applications/:id` | 查询应用详情 |
| GET | `/applications` | 应用列表（支持筛选+分页） |
| PUT | `/applications/:id` | 更新应用 |
| DELETE | `/applications/:id` | 删除应用 |

列表查询参数：
- `key`（可选）
- `name`（可选）
- `status`（可选，`active/inactive`）
- `page`（可选，>=1，默认 1）
- `page_size`（可选，>=1，默认 20，最大 100）

列表响应结构：
- `data: Application[]`
- `page: int`
- `page_size: int`
- `total: int64`

错误码约定：
- `400` 参数错误
- `404` 资源不存在
- `409` 唯一键冲突
- `500` 服务内部错误

## 7. 数据库与持久化需求
### 7.1 存储支持
- 默认 `mysql`
- 可切换 `sqlite`

### 7.2 建表与初始化
- 启动阶段执行 `InitSchema`
- 自动创建 `applications` 表
- `app_key` 必须唯一（MySQL: `UNIQUE KEY`，SQLite: `UNIQUE`）

### 7.3 分页查询策略
- SQL 层先执行 `COUNT(1)` 获取总数
- 再执行 `LIMIT + OFFSET` 获取当前页
- 按 `created_at DESC` 排序

## 8. 配置需求
配置来源：
- 配置文件（默认 `configs/config.local.json`）
- 环境变量覆盖（如 `APP_CONFIG_FILE`、`DB_DRIVER`、`MYSQL_DSN`）

数据库配置关键项：
- `database.driver`
- `database.mysql_dsn`
- `database.sqlite_path`
- 连接池参数（`max_open_conns` 等）
- 启动检查参数（`startup_max_retries`、`startup_retry_interval_sec`、`ping_timeout_sec`）

## 9. 启动与运行需求
应用启动需完成：
- 配置加载
- DB 连接建立
- MySQL 启动重试连通性检查
- 表结构初始化
- 路由注册
- HTTP 服务启动
- 支持优雅停机（SIGINT/SIGTERM）

## 10. 文档与可观测性
- Swagger 文档可生成并通过 `/swagger/*any` 访问
- 启动日志需输出监听地址、环境、数据库驱动

## 11. 验收标准（v0.0.1）
- `applications` 的增删改查接口可用
- `GET /applications` 分页参数生效并返回 `total/page/page_size`
- MySQL 与 SQLite 两种驱动均可启动
- MySQL 不可达时，按配置重试后明确失败退出
- `/healthz` 可返回 200
- Swagger 页面可访问且接口契约与实现一致
