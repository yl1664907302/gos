# 后端需求 v0.0.5（基于 v0.0.4 增量）

## 1. 文档信息
- 文档版本：`v0.0.5`
- 基线日期：`2026-03-13`
- 上一版本：`/Users/lingyunxieqing/Desktop/gos/docs/后端/后端需求v0.0.4.md`
- 本版主题：`发布执行可观测性与取消联动增强`

## 2. 版本目标
在 `v0.0.4` 发布单模型与执行接口基础上，补齐“执行中可追踪、失败可读、取消可联动”的后端能力，重点覆盖：
- Jenkins 结果轮询回写
- 发布日志流式输出（SSE）
- 取消发布时立即向 Jenkins 发送中断
- 执行失败错误信息规范化回显
- 发布单字段补充与兼容迁移

## 3. 本版增量范围
包含：
- 发布单执行状态跟踪任务（定时任务）
- `GET /release-orders/{id}/logs/stream` 日志流接口
- 发布取消与 Jenkins 队列/构建中断联动
- Jenkins 错误消息提取与压缩
- `release_order` 字段与状态兼容迁移

不包含：
- ArgoCD 执行链路
- 审批、灰度、回滚策略
- 多租户权限体系

## 4. 当前实现进度（2026-03-13）

### 4.1 发布单数据模型与迁移
- `release_order` 已新增并落库字段：`son_service`
- `release_order.status` 已统一默认值为 `pending`
- 已增加历史数据兼容修复：将空状态与错误拼写 `pengding` 修正为 `pending`
- MySQL/SQLite 双驱动均已覆盖上述迁移逻辑

### 4.2 发布执行与取消能力
- 已支持发布执行接口：`POST /release-orders/{id}/execute`
- 执行前校验：仅允许 `pending` 状态，且当前仅支持 Jenkins 绑定
- 取消接口：`POST /release-orders/{id}/cancel`
- 当发布单处于 `running` 时，取消会尝试：
- 取消 Jenkins 队列项（`cancelQueue`）
- 停止 Jenkins 构建（`stop`）
- 同时将 Jenkins 相关执行步骤从 `running/pending` 收敛为已结束状态，避免页面长期“执行中”

### 4.3 Jenkins 结果回写（定时追踪）
- 已新增发布追踪用例：`TrackReleaseExecution`
- 已接入启动定时任务，轮询 `running` 发布单并同步 Jenkins 队列/构建状态
- 当 Jenkins 返回终态时，后端会回写步骤与发布单状态（`success/failed/cancelled`）
- 当前默认轮询间隔：`10s`（`jenkins.release_track_interval_sec`）

### 4.4 发布日志流式输出（SSE）
- 已新增接口：`GET /release-orders/{id}/logs/stream?start=<offset>`
- 已支持事件类型：
- `status`（状态提示）
- `log`（日志块）
- `done`（执行结束）
- `error`（流式错误）
- 已支持 Jenkins progressive log（按 offset 增量拉取）

### 4.5 Jenkins 客户端增强
- 已支持：
- `GetBuildConsoleText`（渐进日志）
- `AbortQueueItem`（取消队列）
- `AbortBuild`（终止构建）
- 已增强 Jenkins 错误体解析：
- 从 HTML 页面中提取关键错误（如参数非法）
- 返回更短、更可读的后端错误消息

### 4.6 启动与路由 wiring
- `cmd/server/main.go` 已完成：
- 日志流用例注入
- 发布追踪任务注入
- 相关 handler 注入
- `ReleaseOrderHandler` 已注册日志流路由与执行/取消路由
