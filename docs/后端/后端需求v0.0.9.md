# 后端需求文档 v0.0.9（基于 v0.0.8 增量）

## 1. 文档信息
- 文档版本：`v0.0.9`
- 基线日期：`2026-03-18`
- 上一版本：`/Users/lingyunxieqing/Desktop/gos/docs/后端/后端需求v0.0.8.md`
- 本版主题：`回滚建单链路 + 发布单取值进度增强`

## 2. 本版目标
在 `v0.0.8` 已完成的 `ArgoCD/GitOps` 接入基础上，补齐两条核心链路：
- 平台支持基于应用最近一次成功发布记录创建回滚发布单
- 发布详情里的 `取值进度` 要覆盖系统内置字段，不再只依赖模板显式映射项

## 3. 范围
包含：
- 回滚发布单创建接口
- 发布单新增来源字段
- 回滚记录可追踪
- 内置字段自动进入取值进度
- `CD=ArgoCD` 时内置字段复用视图

不包含：
- 回滚自动执行
- 回滚审批链
- 回滚差异对比
- GitOps 回滚 commit 自动生成专属流程

## 4. 数据模型
### 4.1 release_order
新增字段：
- `previous_order_no`

含义：
- 普通发布单：空字符串
- 回滚发布单：记录来源发布单号

### 4.2 迁移要求
启动时自动迁移：
- MySQL：`ALTER TABLE release_order ADD COLUMN previous_order_no VARCHAR(64) NOT NULL DEFAULT ''`
- SQLite：`ALTER TABLE release_order ADD COLUMN previous_order_no TEXT NOT NULL DEFAULT ''`

不要求回填历史数据。

## 5. 回滚建单
### 5.1 接口
- `POST /applications/{id}/release-orders/rollback`

### 5.2 权限
沿用应用级发布权限：
- `release.create`

### 5.3 回滚来源规则
按应用维度查询：
- 最近一次 `status=success` 的发布单
- 按 `created_at desc`
- 取第一条

若不存在：
- 返回 `400`
- 错误信息：`当前应用暂无可回滚的成功发布单`

### 5.4 回滚建单行为
平台自动：
1. 读取来源发布单
2. 读取来源发布单参数快照
3. 复用：
   - `application_id`
   - `template_id`
   - 参数快照
4. 创建一张新的待执行发布单
5. 写入：
   - `previous_order_no = 来源发布单号`
   - `remark = 回滚自发布单 <order_no>`
   - `creator_user_id = 当前用户`
   - `triggered_by = 当前用户展示名`

### 5.5 设计原则
回滚行为只做：
- `重新建单`

当前版本不做：
- 自动执行

原因：
- 把“回滚建单”和“立即触发执行”拆开，风险更低，审计更清晰。

## 6. 取值进度增强
### 6.1 问题修正
旧逻辑只展示：
- 模板里显式勾选并映射的标准 Key

这会导致：
- 系统内置字段即使是平台必备输入
- 只要未在模板中显式映射
- 就不会出现在 `取值进度`

### 6.2 新规则
`取值进度` 现在分两部分生成：
1. 模板显式参数
2. 平台启用中的内置字段

内置字段规则：
- 不依赖模板是否勾选
- 不依赖用户是否手工映射
- 只要是启用中的 `builtin=true` 标准字段，就进入进度计算

### 6.3 取值来源优先级
后端按以下优先级解析取值：
1. `release_order_param` 参数快照
2. 执行器推导值
   - Jenkins `image_version` -> `BUILD_NUMBER`
3. 发布单摘要字段
   - `env` / `env_code` -> `release_order.env_code`
   - `branch` / `git_ref` -> `release_order.git_ref`
   - `project_name` -> `release_order.son_service`
   - `image_tag` / `image_version` -> `release_order.image_tag`

### 6.4 ArgoCD 复用规则
当模板 `CD=ArgoCD` 时：
- `CD` 侧的内置字段优先展示“沿用 CI”的结果
- 若 `CI` 已解析出值：
  - `CD` 直接展示同值
- 若 `CI` 尚未取值：
  - `CD` 状态为等待
- 若 `CI` 失败：
  - `CD` 提示无法继续复用

### 6.5 说明
这套规则的目标不是增加新的持久化表，而是：
- 基于现有模板
- 基于参数快照
- 基于执行状态
- 实时算出“当前字段值已经走到哪一步”

## 7. 领域与用例调整

- `PreviousOrderNo`

### 7.2 ReleaseOrderManager
新增能力：
- `CreateRollbackByApplication(...)`

职责：
- 查询最近成功单
- 复用参数快照
- 创建回滚发布单

- 不再只依赖模板参数集合
