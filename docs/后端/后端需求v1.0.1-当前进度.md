## 6. 当前仍需继续补强的后端点

### 6.1 发布前预演
当前缺口：
- 预检能力存在，但仍建立在“已有发布单”的前提下
- 用户建单前无法拿到完整执行预期

建议补齐：
- 新增 `Preview Release Plan` 接口
- 在不落库、不触发外部系统的前提下，计算：
  - 审批命中结果
  - 执行单元计划
  - 参数最终取值
  - Hook 执行预期
  - GitOps 修改目标
  - 并发与排队预判

### 6.2 失败后续跑
当前缺口：
- 失败单更多依赖新建或回滚，缺少“从失败点续跑”模型

建议补齐：
- 失败步骤继续执行
- 从成功快照复用结果并续跑剩余阶段

### 6.3 发布计划与执行解耦
当前缺口：
- 创建发布单与计算执行计划仍有一定耦合

建议补齐：
- 抽出独立 `Release Plan Builder`
- `preview` 与 `create` 共用

### 6.4 可观测与审计聚合
当前缺口：
- 已有日志、步骤、Hook、Agent 记录，但缺少统一的全局审计视图

建议补齐：
- 发布时间线
- 全局操作审计
- 统一异常归因模型
