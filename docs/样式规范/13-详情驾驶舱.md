# 详情驾驶舱

## 适用场景
- 发布单详情
- 执行流详情
- 巡检详情
- 需要同时展示状态、进度、日志、上下文的工程页

当前基线页面：
- `/Users/lingyunxieqing/Desktop/gos/frontend/src/views/release/ReleaseOrderDetailView.vue`

## 设计目标
- 让研发用户一眼看到“现在卡在哪”
- 优先展示结论，而不是先展示完整明细表
- 减少长页面纵向滚动
- 把“实时进度”和“排障上下文”放在同一视野内

## 推荐结构
### 1. 顶部总览卡
- 放在页面头部卡片之后
- 展示：
  - 编号
  - 应用
  - 环境
  - Git 版本
- 必须提供一个“当前卡点/当前进展”提示区
- 总览区允许比普通详情页更高的视觉权重
- 若状态已经通过右侧整体进度大视图清晰表达，不必在左侧重复展示同一状态字。

### 2. 主副双栏
- 推荐桌面端采用 `主内容区 + 右侧上下文区`
- 主内容区优先放：
  - 执行时间线
  - 阶段进度
  - 实时日志
- 右侧上下文区优先放：
  - 执行单元状态
  - 发布上下文
  - 取值进度
  - 命中的 ArgoCD / GitOps / 集群信息

### 3. 底部信息降权
- 基础信息
- 参数快照
- 低频排障字段

这类信息默认进入折叠面板，不抢主视线。

## 交互规范
### 当前卡点
- 必须直接显示当前失败步骤或当前运行步骤
- 文案应采用：
  - `步骤名 + 错误/状态摘要`
- 不允许只显示状态色，不给出步骤语义

### 进度与日志
- 阶段进度和实时日志推荐放到同一卡片内，通过 `Tab` 切换
- 避免把日志、阶段、步骤拆成连续长区块，导致用户上下滚动找关联信息

### 执行单元
- `CI` 与 `CD` 作为独立执行单元卡片展示
- 卡片中至少展示：
  - 执行器类型
  - 当前状态
  - 开始时间
  - 结束时间

## 视觉语义
### 状态色
- 成功：绿色
- 执行中：蓝色
- 待执行：灰黄或中性色
- 失败：红色

### 样式原则
- 总览卡、当前卡点、时间线当前节点，允许更高对比度
- 但颜色仍应服从 `/Users/lingyunxieqing/Desktop/gos/docs/样式规范/08-状态反馈.md`
- 不使用大面积高饱和底色铺满整页
- 右侧大视图中的图标应起到“状态仪表”作用，避免做成突兀的大色块按钮。

## 信息优先级
### 第一优先级
- 当前状态
- 当前卡点
- 当前正在执行的阶段

### 第二优先级
- 执行单元
- 取值进度
- 命中的部署上下文

### 第三优先级
- 模板 ID
- 创建者
- 更新时间
- 参数快照

## 响应式要求
- 窄屏下双栏自动回落成单列
- 总览卡中的事实项允许折行，不允许压缩成难以阅读的超窄列
- 右侧上下文区在移动端应落到主内容区下方，不单独保留固定侧栏

## 禁用项
- 禁止把详情页继续做成纯表格堆叠页
- 禁止把所有信息常驻展开，导致滚动距离过长
- 禁止把失败原因藏在底部日志区，顶部只显示“失败”
- 禁止在研发排障页面中过度装饰化，牺牲信息密度和可扫描性

## 落地建议
- 先抽象总览卡、上下文卡、时间线卡的样式基类
- 后续类似页面优先复用该布局，不重复发明新的详情页骨架
- 新增执行/巡检/部署类页面时，优先判断是否应套用本规范
