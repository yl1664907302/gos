# 驾驶舱风格

## 适用范围
- 平台主框架
- 列表页
- 发布页
- 详情页
- 登录页中的登录卡

## 风格目标
- 不是“炫技式大屏”，而是工程中台里的控制台气质。
- 重点是秩序感、状态感、信息层级，而不是夸张装饰。
- 要让研发用户进入页面后，第一眼感到稳、清楚、可信。

## 核心特征
### 1. 深色控制壳
- 左侧菜单使用深石板蓝作为主底色。
- 菜单承担“控制中枢”角色，允许比右侧内容区更重。
- 右侧内容区保持明亮，形成“暗壳 + 亮内容”的对比。

### 2. 亮面内容卡
- 业务内容仍以白底或浅亮面板为主。
- 卡片通过轻阴影、浅边框和高光来建立层级。
- 禁止整页使用深色卡片堆叠，否则会让信息变暗、难读。

### 3. 强状态语义
- 成功：亮绿
- 失败：亮红
- 执行中：亮蓝
- 待执行/中性：灰蓝或暖中性色
- 状态色应服务于步骤、结果、提示，不应用作大面积装饰背景。

### 4. 控制面板感
- 顶部总览卡、右侧进度卡、当前关注区可以带少量高对比度视觉。
- 图标、胶囊、边框、分隔线统一使用同一套石板蓝系与状态色。
- 控制感来自布局和状态，不来自堆砌说明文案。

## 色彩角色
### 主控色
- `--color-dashboard-900`
- `--color-dashboard-800`

用途：
- 左侧菜单
- 表头
- 主按钮
- 重点控制块

### 内容色
- `--color-bg-page`
- `--color-bg-card`
- `--color-bg-subtle`

用途：
- 页面背景
- 卡片背景
- 筛选面板
- 表单容器

### 状态色
- `--color-success`
- `--color-warning`
- `--color-danger`
- `--color-info`

用途：
- Tag
- Alert
- 步骤图标
- 状态胶囊

## 页面落地规则
### 壳层
- 左侧导航固定在页面左侧。
- 用户信息和退出入口固定在左下角。
- 顶部全局 header 不再承担导航职责，避免形成两套控制中枢。

### 标题区
- 标题要稳，副标题要短。
- 不要再使用“大段说明文案 + 英文眉标”堆气质。
- 标题区负责建立页面信心，不负责解释全部业务。

### 查询区
- 优先白底或透明轻面板。
- 外层大白卡可以去掉，保留内部筛选结构。
- 查询与重置按钮使用深石板蓝主按钮体系，不使用默认浅蓝 hover。

### 表格
- 表头优先使用 `--color-dashboard-900`。
- 操作列使用石板蓝文字按钮，危险操作保留红色。
- 表格区可以去掉额外外壳，让页面更轻。

### 提示块
- 使用浅亮的 success / warning / error / info 面板。
- 禁止出现大片发暗提示块。
- 提示块的作用是“说明状态”，不是抢页面主视觉。

### 登录框
- 登录页允许使用深色背景，但应把设计重点放在登录卡本身。
- 登录卡需精致、克制、可快速聚焦。
- 不要把登录页做成宣传页或信息大屏。

## 禁用项
- 禁止把“驾驶舱”理解成大量堆砌发光、渐变、说明文案。
- 禁止整页大面积深色卡片，导致文本可读性下降。
- 禁止深色壳层和右侧亮内容区没有主次区分。
- 禁止一个页面里同时出现多套蓝色体系。
- 禁止让默认 Ant 淡蓝 hover 残留在主业务页。

## 落地建议
- 所有新页面先判断：它属于壳层页、列表页、详情页还是表单页。
- 先套壳层和色板，再调局部组件，不要一页一页重新发明风格。
- 如果一个页面“看起来很花”，优先删除装饰，再回到状态、层级和节奏本身。
