# 状态与反馈

## 统一反馈组件
- 成功：`message.success`
- 失败：`message.error`
- 提醒：`message.info` / `message.warning`

## 错误文案
- 失败场景优先展示后端原始可读错误
- 前端默认兜底文案使用业务动作语义（例如“加载失败”“更新失败”）

## 视觉语义
- 危险色：`#ff4d4f`
- 危险图标类：`.danger-icon`
- 状态标签统一使用 `a-tag`

## 辅助说明提示
- 非阻断辅助说明优先使用点击式提示气泡，不使用常驻大块提示
- 具体触发器、气泡内容和禁用项见：`/Users/lingyunxieqing/Desktop/gos/docs/样式规范/组件UI规范/提示气泡样式规范.md`
