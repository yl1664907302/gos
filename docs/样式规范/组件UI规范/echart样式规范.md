# EChart 样式规范

## 适用范围
- 用于发布页、工作台、统计概览等需要图表表达状态分布的模块
- 当前基线实现：`/Users/lingyunxieqing/Desktop/gos/frontend/src/views/release/ReleaseOrderListView.vue`
- 典型选择器：`.overview-chart-panel`、`.overview-chart-canvas`

## 设计定位
- 图表卡片是统计概览组件，不是查询结果列表的附属说明
- 图表默认展示全局统计口径，除非需求明确说明按条件联动
- 图表标题必须说明统计口径，例如“全部发布单状态分布”
- 小字说明不带句号，例如“统计口径：汇总全部发布单状态数量”

## 数据口径
- 全局概览图表不跟随条件查询、状态查询、环境筛选变动
- 查询条件只影响列表和相关关注信息，不影响全局图表
- 统计 API 只传最小必要参数，例如 `page: 1`、`page_size: 1`
- 不把 `activeQuery`、表格分页、当前页数据作为 EChart 统计来源
- 总数已经在卡片右上角展示时，柱状图不再额外展示“总数”柱

## 卡片外壳
- 使用深色仪表卡风格，避免浅色普通卡片
- 推荐最小高度：桌面端 `236px`
- 推荐圆角：`20px`
- 推荐内边距：桌面端 `18px`，移动端 `16px`
- 使用弱边框：`rgba(71, 85, 105, 0.4)`
- 使用多层背景：
  - 右上绿色径向光
  - 左上蓝色径向光
  - 深色纵向渐变底
- 顶部可以加 1px 彩色光线作为精致感，但不可喧宾夺主
- 阴影应偏深、低透明，突出悬浮层级

## 信息结构
- 顶部结构固定为左文案、右统计胶囊
- 左侧依次展示：
  - 分类标签，例如“发布统计”
  - 主标题，例如“全部发布单状态分布”
- 右侧展示总数胶囊，例如“共 225 条”
- 图表区域位于标题下方
- 底部展示统计口径小字

## 字体层级
- 分类标签：`12px`，`font-weight: 700`，字距 `0.08em`
- 主标题：桌面端 `20px`，移动端 `18px`，`font-weight: 800`
- 总数胶囊：`12px`，`font-weight: 700`
- 底部小字：`12px`，`line-height: 1.6`

## 图表尺寸
- 桌面端画布高度：`142px`
- 移动端画布高度：`132px`
- `grid.top` 使用 `16`
- `grid.bottom` 使用 `0`
- `grid.left` 和 `grid.right` 使用紧凑值，例如 `8`
- `containLabel` 必须开启
- 柱宽推荐 `34%`，避免粗重拥挤

## EChart 配置
- 推荐使用柱状图展示状态分布
- 推荐标签顺序：`待处理`、`执行中`、`失败`、`成功`
- 推荐颜色：
  - 待处理：`rgba(148, 163, 184, 0.68)`
  - 执行中：`#60a5fa`
  - 失败：`#f87171`
  - 成功：`#34d399`
- tooltip 使用深色浮层，背景 `rgba(2, 6, 23, 0.92)`
- x 轴文字使用浅色低对比，强调图表主体
- y 轴只保留弱分割线，不显示轴线和刻度
- `minInterval` 使用 `1`，避免数量类图表出现小数刻度

## 响应式
- `max-width: 1024px` 时概览区域改为单列
- 图表卡片单列后仍保持 `236px` 最小高度
- `max-width: 768px` 时图表卡片最小高度降为 `210px`
- 移动端标题字号降为 `18px`
- 移动端画布高度降为 `132px`

## 禁用项
- 不使用默认 EChart 白底图表
- 不使用紫色默认渐变
- 不把“总数”重复画进柱状图
- 不跟随查询条件自动刷新全局图表
- 不使用当前页表格数据推导全局状态分布
- 不在小字说明末尾加句号
