# 一般卡片风格

## 适用范围
- 应用上下文中的绑定、配置、摘要、阶段入口类页面
- 典型页面：`/applications/:id/pipeline-bindings`
- 基线实现：`/Users/lingyunxieqing/Desktop/gos/frontend/src/views/application/ApplicationPipelineBindingView.vue`

## 风格定位
- 一般卡片属于浅色业务卡，不使用应用工作台的深色控制卡语言
- 它的职责是“看状态 + 进操作”，不是“做展示型装饰”
- 页面里可以使用卡片网格，但不因此切换到深色工作台主题

## 卡片外壳
- 使用白到浅灰的轻渐变或纯浅色面板
- 使用浅边框 + 轻阴影
- 允许非常弱的局部高光，但不允许整卡状态底色
- 不使用顶部颜色线
- 不使用大面积发光边、整卡纯色块、驾驶舱式装饰

## 模块图标
- 模块图标统一使用共享浅色壳层
- CI/CD 仅通过图标色、轻描边区分
- 不使用整块纯色底图标
- 空态不额外放装饰图标

## 信息结构
### 已绑定卡片
- 信息层次固定为三段：
  - 头部：模块图标、标题、状态 tag、更新时间小标签、短说明
  - 内容：摘要信息 + 关键事实字段
  - 底部：独立操作工具栏
- 卡片信息必须收敛，只保留：
  - 名称
  - 一行元信息
  - 主标识，例如 `pipeline_id`
  - 更新时间
- 更新时间作为小标签展示在“已绑定”状态气泡旁
- 不在详情事实区重复放一行“更新时间”

### 未绑定卡片
- 未绑定卡片与已绑定卡片保持同排等高
- 只保留标题和一句说明
- 不放装饰图标
- 不放内联 CTA

## 操作区
- 操作按钮统一放在卡片 footer
- 桌面端默认一行等分排列
- 小屏再折行
- 卡片内部不放“新增 xx 绑定”主入口
- 新增入口统一留在页面标题区

## 页面结构
- 优先使用标题区 + 双卡或多卡模块区
- 不回退到“查询大卡 + 表格大卡”的旧结构
- 适合展示阶段入口、绑定摘要、轻量配置入口

## 使用边界
- 一般卡片不承担应用工作台那种全局状态控制职责
- 不直接复用应用工作台深色卡按钮和深色卡外壳
