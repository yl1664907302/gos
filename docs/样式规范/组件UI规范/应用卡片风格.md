# 应用卡片风格

## 适用范围
- 页面：`/applications`
- 组件角色：应用工作台主卡片
- 基线实现：`/Users/lingyunxieqing/Desktop/gos/frontend/src/views/application/ApplicationListView.vue`

## 风格定位
- 这是平台里少数允许使用深色控制台风格的卡片类型
- 卡片承担“状态浏览 + 快速操作 + 环境切换”三类职责
- 它不是普通业务信息卡，也不是表格替代品
- 同页面里如果同时存在浅卡和深卡，应用卡片必须是主视觉

## 卡片角色
- 每张卡片代表一个应用的当前工作面板
- 折叠态负责快速浏览和主操作
- 详情态负责环境维度的切换与发布上下文
- 卡片内部表达的是“当前应用状态”，不是静态资料展示

## 结构规范
### 折叠态
- 默认展示折叠态
- 必须包含：
  - 项目名
  - 负责人
  - 应用中文名称
  - 应用 Key
  - 最近发布单号
  - 当前发布状态
  - 操作按钮：`发布`、`详情/查单`、`配置`
- `详情` 与 `查单` 互斥：
  - 有环境视图数据时显示 `详情`
  - 无环境视图数据时显示 `查单`

### 详情态
- 不展开为跨卡大面板
- 点击 `详情` 后只在卡片内部切换视图
- 当前保留内容：
  - 当前环境标题
  - 生效发布单气泡
  - 最近发布单气泡
  - 底部按钮：`切换`、`发布单`、`返回`
- 当仅有一个环境时，不显示 `切换`
- 当无环境数据时，展示空态，不做禁用详情按钮

## 视觉规范
- 卡片使用深色控制台风格
- 折叠态和详情态保持同一设计语言
- 深色卡可使用更强的层次、发光和轻动效
- 运行中应用允许整卡进入发布动效态
- 动效只能由真实 `runningCount > 0` 触发
- 不允许为了“看起来有状态”常亮伪动效

## 按钮规范
- `发布`：主操作按钮
- `详情/查单`：次主操作按钮
- `配置`：管理操作入口
- 普通用户可以看到 `配置`，但不可执行
- 管理员或具备应用管理权限的用户可点击 `配置`
- 卡片内按钮属于工作台场景按钮，不直接外溢到普通表单页和列表页

## 使用边界
- 深色卡只用于真正需要控制感和状态感的业务块
- 不得把应用工作台深色卡样式直接套到绑定、配置、摘要、空态承托块
- 应用上下文里的普通配置卡，统一改用：
