# 当前关注组件样式规范

## 适用范围
- 用于页面顶部或统计区右侧的关键状态提醒卡
- 当前基线实现：`/Users/lingyunxieqing/Desktop/gos/frontend/src/views/release/ReleaseOrderListView.vue`
- 典型选择器：`.overview-spotlight`

## 设计定位
- 当前关注组件用于提示当前最需要用户注意的信息
- 它是概览区的右侧信息卡，不承担表格筛选、批量操作或复杂详情职责
- 默认关注应展示“最新发布”，避免默认固定为失败或执行中
- 当用户选择状态筛选时，关注内容可以切换为当前状态聚焦

## 内容结构
- 卡片内容分为三段：
  - 顶部：标签、主标题、辅助说明、状态图标
  - 中部：最新两个单号
  - 底部：最近刷新时间与自动轮询状态
- 标签固定为“当前关注”
- 主标题使用短文本，例如“最新发布”“发布失败”“执行中”
- 辅助说明单独一行，不和主标题重复
- 单号必须一行一个，最多展示两个
- 单号必须可点击跳转到发布单详情
- 小字说明不带句号

## 卡片外壳
- 使用深色渐变卡片，与 EChart 深色统计卡形成同组视觉
- 推荐最小高度：桌面端 `236px`
- 推荐圆角：`20px`
- 推荐内边距：桌面端 `18px`，移动端 `16px`
- 使用 `space-between` 让短内容在卡片内自然分布
- 使用右上径向高光 + 深蓝渐变
- 文字颜色使用 dashboard token，不直接回退到普通正文色

## 状态图标
- 图标固定在右上角
- 图标容器推荐尺寸：`64px * 64px`
- 图标容器圆角：`20px`
- 图标字号：`28px`
- 图标容器使用半透明白底、浅边框、内高光和轻阴影
- 图标区域要和卡片比例协调，不能过小
- 不同状态只改变图标语义色，不改变卡片整体结构

## 文案层级
- 标签：`12px`，字距 `0.08em`，使用弱化标签色
- 主标题：`24px`，`font-weight: 800`，`line-height: 1.2`
- 辅助说明：`13px`，`line-height: 1.55`
- 单号标签：`12px`
- 单号胶囊：`12px`，`font-weight: 700`
- 底部元信息：`12px`

## 单号链接
- 单号链接容器使用纵向布局
- 链接间距推荐 `8px`
- 单个单号使用胶囊样式
- 胶囊最小高度 `28px`
- 胶囊内边距 `0 10px`
- 胶囊背景使用 `rgba(255, 255, 255, 0.1)`
- hover 时背景略加亮并上移 `1px`
- 长单号允许在自身最大宽度内展示，不要挤压状态图标

## 布局避让
- 主标题、标签、辅助说明需要给右上图标预留空间
- 推荐右侧避让：
  - 标签 `padding-right: 92px`
  - 主标题 `padding-right: 92px`
  - 辅助说明 `padding-right: 88px`
- 单号区域右侧保留 `18px`，避免贴边

## 响应式
- `max-width: 1024px` 时跟随概览区变为单列
- `max-width: 768px` 时最小高度降为 `198px`
- 移动端主标题字号降为 `22px`
- 移动端仍保持单号一行一个

## 行为规则
- 默认状态不加状态过滤，只取最新发布单
- 用户选择状态后，单号列表按当前状态取最新两条
- 单号列表刷新可以跟随查询条件和状态筛选
- EChart 全局统计不跟随当前关注的筛选行为
- 最近刷新时间由页面统一维护，不在组件内拼接重复标签

## 禁用项
- 不把主标题和辅助说明写成一段长句
- 不重复展示刷新文案
- 不把两个单号横向挤在同一行
- 不使用过小图标
- 不在卡片中放复杂表格、表单或批量操作
- 不在小字说明末尾加句号
