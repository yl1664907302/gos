# 按钮右侧弹出视图设计规范

## 1. 目的
- 统一“点击按钮或关键文本后，从页面右侧弹出详情视图”的 UI 与交互。
- 避免把详情信息塞回卡片、表格行或新增独立详情页，保持列表上下文不断流。
- 本规范以“管线绑定查看抽屉”和“应用名称点击后的应用信息抽屉”为基线。

## 2. 适用范围
- 前端技术栈：`Vue 3 + Ant Design Vue`
- 组件范围：`a-drawer` 承载只读详情、轻量上下文说明、多值配置展示
- 触发入口：
  - 表格或卡片中的 `查看` 按钮
  - 标题栏辅助按钮
  - 关键业务对象名称，例如应用卡片中的应用名称
- 不适用于新增、编辑、映射配置类表单；这类场景继续使用 `按钮弹窗风格.md`。

## 4. 触发入口规范

### 4.1 按钮触发
- 常规详情入口使用明确按钮文案：`查看`、`详情`、`查看详情`。
- 表格操作列优先使用文字按钮，不使用实心主按钮。
- 卡片 footer 或工具栏中，详情按钮应与同区域按钮保持同源视觉，不单独发明颜色。
- 按钮只负责打开右侧抽屉，不跳转详情页，不在当前卡片内展开大块详情。

### 4.2 关键文本触发
- 当业务对象名称本身就是详情入口时，可以不新增“信息”按钮。
- 文本触发必须具备可点击提示：
  - `cursor: pointer`
  - hover 状态
  - focus 可见
  - 保留原文本层级，不做成胶囊按钮
- 示例：应用卡片中点击应用名称打开“应用信息”抽屉，不再额外增加“信息”按钮。

### 4.3 禁止项
- 禁止同时提供“名称点击”和右侧“信息”按钮两个重复入口。
- 禁止点击名称跳详情页，而按钮打开抽屉，造成同一对象两套详情路径。
- 禁止在卡片内部做大面积折叠详情，破坏工作台卡片高度与列表节奏。

## 5. 抽屉结构规范

### 5.1 组件基线
```vue
<a-drawer :open="visible" title="详情标题" width="640" @close="closeDrawer">
  <a-skeleton v-if="loading" active :paragraph="{ rows: 8 }" />
  <a-descriptions v-else-if="detailData" :column="1" bordered>
    <a-descriptions-item label="字段名">
      字段值
    </a-descriptions-item>
  </a-descriptions>
</a-drawer>
```

### 5.2 宽度
- 标准详情：`width="640"`
- 信息密度较高或含长文本：`width="680"`
- 执行参数、日志、JSON 类详情：`width="720"` 起
- 禁止为普通详情抽屉设置超过 `760` 的宽度。

### 5.3 标题
- 标题使用业务对象类型：`应用信息`、`绑定详情`、`字段说明`。
- 标题不写副标题，不放多按钮工具栏。
- 抽屉标题区域保持轻量，主要信息放入内容区。

## 6. 内容布局规范

### 6.1 基础字段
- 详情主体优先使用 `a-descriptions`。
- 单列：`:column="1"`。
- 边框：`bordered`。
- 字段顺序固定为“身份信息 -> 归属信息 -> 状态信息 -> 配置信息 -> 时间信息”。
- 空值统一展示 `-`，不要留空。

### 6.2 多值字段
- 多值字段不要用逗号拼接。
- 多值字段不要堆叠大量 Tag。
- 多值字段统一在 `a-descriptions-item` 内使用紧凑小表格。
- 小表格应保留表头，便于用户理解列含义。
- 超过 5 条时，表格容器限制高度并内部滚动。

### 6.3 GitOps 映射展示
- 展示列：`环境`、`分支`。
- 排序优先级：`dev -> test -> prod -> 其他`。
- 其他环境按环境名升序。
- 缺失值使用 `-`。

```text
环境        分支
dev         develop
test        release/2.4.3
prod        main
```

### 6.4 发布分支展示
- 展示列：`名称`、`分支`。
- 顺序保持后端或表单配置顺序。
- 缺失值使用 `-`。

```text
名称        分支
默认分支    main
测试分支    release/2.4.3
热修分支    hotfix/xxx
```

### 6.5 空态
- 多值字段为空时，在对应字段位置显示文案：
  - `当前未配置映射`
  - `当前未配置发布分支`
- 不额外渲染空表格。

## 7. 小表格视觉规范

### 7.1 容器
- 容器圆角：`12px`
- 边框：`1px solid #e6ebf5`
- 背景：`#ffffff`
- 最大高度：`220px`
- 溢出：`overflow: auto`

### 7.2 表格
- 宽度：`100%`
- `border-collapse: collapse`
- `table-layout: fixed`
- 字号：`13px`
- 单元格内边距：`9px 12px`
- 文本允许换行，避免长分支名撑破抽屉。

### 7.3 表头
- 表头背景：`#f8fafc`
- 表头文字：`#64748b`
- 字重：`700`
- 表头建议 sticky，滚动时保持列语义可见。

### 7.4 首列
- 首列宽度建议 `34%`。
- 首列字重略高，用于突出环境名或分支名称。

## 8. 遮蔽与层级
- 详情抽屉使用 `a-drawer` 默认从右侧进入。
- 普通详情抽屉允许覆盖右侧内容区，不覆盖左侧导航是优先目标。
- 若页面已有内容区 inset 计算能力，应复用和弹窗一致的遮蔽偏移逻辑。
- 遮蔽层保持轻量，不使用重黑蒙层。
- 抽屉关闭后必须释放点击，不残留遮罩。

## 9. 状态与交互

### 9.1 打开
- 点击触发入口后立即打开抽屉。
- 需要异步加载时，抽屉先打开并显示 `a-skeleton`。
- 已有完整行数据时，可以直接渲染详情，不额外请求。

### 9.2 关闭
- 点击关闭按钮、遮罩或按 `Esc` 都应关闭抽屉。
- 关闭时保留或清理选中数据都可以，但重新打开必须以当前点击对象为准。
- 如果详情依赖异步请求，关闭后应避免旧请求回写当前抽屉。

### 9.3 刷新
- 抽屉内只做查看，不承载编辑保存。
- 若需要编辑，提供独立编辑入口，进入编辑页或编辑弹窗。
- 详情抽屉内不放大面积操作按钮，避免从查看视图变成操作面板。

## 10. 按钮适配

### 10.1 表格操作按钮
- 使用文字按钮或轻量 link button。
- 文案短：`查看`。
- 不使用主色实心按钮，避免操作列过重。

### 10.2 卡片内入口
- 如果卡片已有明确操作按钮区，详情入口可以放入 footer。
- 如果对象名称已承担详情入口，不再新增“信息”按钮。
- 名称点击必须具备 hover 反馈，用户可感知。

### 10.3 页头按钮
- 页头触发抽屉的按钮复用 `application-toolbar-action-btn` 类视觉体系。
- 与查询、刷新、创建等按钮保持同一高度、圆角、间距。

## 11. 响应式
- 桌面端标准宽度：`640`。
- 窄屏下允许抽屉接近全宽，但内容仍保持单列。
- 小表格在窄屏继续使用横向自适应，不强制拆成卡片。
- 长分支名必须换行，不允许横向撑出抽屉。

## 12. 可访问性
- 可点击文本必须支持键盘焦点。
- 触发按钮要有明确文案，不只依赖图标。
- 表格必须保留 `thead`，不能用纯 `div` 模拟无语义表格。
- 动效遵循系统 `prefers-reduced-motion`，不得依赖动画传达唯一信息。

## 13. 推荐实现模板

```vue
<button class="object-name-trigger" type="button" @click="openInfoDrawer(record)">
  {{ record.name }}
</button>

<a-drawer :open="infoVisible" title="应用信息" width="640" @close="closeInfoDrawer">
  <a-descriptions v-if="selectedRecord" :column="1" bordered>
    <a-descriptions-item label="应用名称">
      {{ selectedRecord.name || '-' }}
    </a-descriptions-item>

    <a-descriptions-item label="GitOps 映射">
      <div v-if="selectedRecord.gitops_branch_mappings?.length" class="info-mini-table-scroll">
        <table class="info-mini-table">
          <thead>
            <tr>
              <th>环境</th>
              <th>分支</th>
            </tr>
          </thead>
          <tbody>
            <tr v-for="item in sortedGitOpsMappings(selectedRecord)" :key="`${item.env_code}-${item.branch}`">
              <td>{{ item.env_code || '-' }}</td>
              <td>{{ item.branch || '-' }}</td>
            </tr>
          </tbody>
        </table>
      </div>
      <span v-else>当前未配置映射</span>
    </a-descriptions-item>
  </a-descriptions>
</a-drawer>
```

## 14. 禁止项
- 禁止新增重复的“信息”按钮。
- 禁止让详情抽屉与详情页同时承担同一信息入口。
- 禁止把多值字段塞进一行纯文本。
- 禁止把多值字段渲染为大量无结构 Tag。
- 禁止把查看抽屉做成编辑弹窗。
- 禁止在抽屉内叠多层白色卡片。
- 禁止为普通详情抽屉使用过重遮罩或深色大背景。
