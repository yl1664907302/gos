# 按钮弹窗风格

## 1. 目的
- 固化当前“编辑类弹窗”已经落地的按钮弹窗风格，避免再次回退到默认 `a-modal` 页脚按钮样式。
- 统一弹窗的 UI 壳层、标题布局、按钮样式、遮蔽范围与内容节奏。
- 本规范以“管线绑定页编辑弹窗”为第一基线。

## 2. 适用范围
- 前端技术栈：`Vue 3 + Ant Design Vue`
- 组件范围：`a-modal` 内承载 `a-form` 的新增、编辑、映射类表单
- 交互风格：按钮放在标题栏右侧，不使用默认 footer

## 4. 结构规范

### 4.1 弹窗组件
- 使用 `a-modal`
- 固定宽度：`:width="760"`
- 关闭图标：`:closable="false"`
- 默认页脚：`:footer="null"`
- 关闭销毁：`:destroy-on-close="true"`
- 关闭后清理：`:after-close="handleFormAfterClose"`
- 必须通过 `wrap-class-name` 提供独立弹窗壳层 class

### 4.2 标题栏
- 使用 `#title` 插槽自定义标题栏
- 标题栏结构固定为左右两端：
  - 左：标题文本
  - 右：保存按钮
- 标题容器使用 `display: flex; justify-content: space-between; align-items: center;`
- 标题文本单独承载大字号和粗字重，不能把大字号直接挂在整个 `ant-modal-title` 容器上

### 4.3 内容区
- 表单使用 `a-form`，`layout="vertical"`
- 必须关闭默认红色星号：`:required-mark="false"`
- 编辑态优先展示：
  1. 只读说明
  2. 当前上下文
  3. 可编辑配置
- 内容分区之间用轻边线分隔，不额外套重卡片

## 5. UI 壳层规范

### 5.1 弹窗外壳
- 弹窗内容壳层使用浅色业务卡背景，不使用默认纯白面板
- 推荐背景：
  - 右上绿色浅光晕
  - 左下蓝色浅光晕
  - 白到浅灰的纵向渐变底
- 推荐边框：`1px solid rgba(255, 255, 255, 0.68)`
- 推荐圆角：`24px`
- 推荐阴影：
  - 外层：`0 32px 90px rgba(15, 23, 42, 0.18)`
  - 内层高光：`inset 0 1px 0 rgba(255, 255, 255, 0.96)`
- 推荐毛玻璃：`backdrop-filter: blur(18px) saturate(180%)`

### 5.2 顶层高光
- 允许在 `ant-modal-content::before` 上增加一层中性高光蒙版
- 高光只做轻量提亮，不做强色渐变或品牌装饰
- 高光层必须 `pointer-events: none`

## 6. 按钮规范

### 6.1 标题栏保存按钮
- 保存按钮放在标题栏右侧，不再放在底部 footer
- 复用页面头部按钮壳层：`application-toolbar-action-btn`
- 可以增加一个局部 class 只做布局约束，例如：
  - `flex: none`
  - `font-size: 14px`
- 按钮文案统一为 `保存`

### 6.2 按钮视觉
- 高度：`42px`
- 圆角：`16px`
- 内边距：`14px`
- 背景：半透明浅色玻璃按钮
- 边框：浅色描边
- 字号：`14px`
- 字重：`700`
- 字色：统一使用深色正文色，推荐 `#0f172a`
- Hover/Focus 态保持同一套深色字色，不单独切成蓝字
- Hover/Focus：只做轻提亮，不改成重色实底
- 禁止页面局部把共享按钮壳层文字覆写成其他主色

### 6.3 禁止项
- 禁止保留默认 `ok-text="保存"` / `cancel-text="取消"` footer 按钮
- 禁止把取消按钮放回弹窗底部形成双操作区
- 禁止在标题栏同时放“大标题 + 副标题 + 多个主按钮”

## 7. 遮蔽规范

### 7.1 遮蔽范围
- 遮蔽层与弹窗 wrap 只覆盖主内容区，不覆盖左侧菜单
- 需要通过 `mask-style` 与 `wrap-props` 同步设置：
  - `left: ${viewportInset}px`
  - `width: calc(100% - ${viewportInset}px)`

### 7.2 遮蔽视觉
- 遮蔽层保持浅遮挡，不允许黑色重蒙层
- 推荐背景：`rgba(15, 23, 42, 0.08)`
- 推荐模糊：`blur(10px)`
- 目标是“看得见页面上下文，但能清楚感知弹窗焦点”

### 7.3 遮蔽交互
- 弹窗关闭后，遮蔽层与 wrap 必须立即释放点击：
  - `pointerEvents: visible ? 'auto' : 'none'`
- 需要监听布局尺寸变化，让遮蔽与 wrap 偏移始终跟随侧栏宽度

## 8. 表单布局规范

### 8.1 说明块
- 弹窗顶部建议放置一个轻量说明块，创建/编辑模式均显示
- 创建模式说明表单用途或关键约束
- 编辑模式说明哪些字段只读、为什么只读
- 说明块只保留左侧竖向强调条，不再包一层提示卡背景
- 说明块、小字提示、辅助说明统一使用短句，不带句号

**强调条样式（推荐）：**
- 宽度 `4px`，圆角 `999px`
- 管理类/任务类表单使用蓝色渐变：
  `background: linear-gradient(180deg, rgba(59, 130, 246, 0.42), rgba(96, 165, 250, 0.16))`
- 绑定类/配置类表单可使用琥珀色渐变（见基线实现中的 `.binding-form-note`）

**CSS 参考：**
```css
.xxx-form-note {
  position: relative;
  padding: 0 0 0 14px;
  color: #64748b;
  font-size: 13px;
  line-height: 1.6;
}

.xxx-form-note::before {
  content: '';
  position: absolute;
  left: 0;
  top: 3px;
  bottom: 3px;
  width: 4px;
  border-radius: 999px;
  background: linear-gradient(180deg, rgba(59, 130, 246, 0.42), rgba(96, 165, 250, 0.16));
}
```

### 8.2 上下文块
- 编辑态允许渲染“当前绑定/当前上下文”只读区域
- 推荐两列网格：`repeat(2, minmax(0, 1fr))`
- 行内用底部虚线分隔，不单独做实心卡片

### 8.3 分区标题
- 分区标题左侧为文字，右侧为一条渐隐分隔线
- 推荐字号：`14px`
- 推荐字重：`700`
- 分区之间使用轻边线与顶部留白拉开节奏

**CSS 参考：**
```css
.xxx-form-panel-title {
  display: flex;
  align-items: center;
  gap: 12px;
  margin-bottom: 14px;
  color: #0f172a;
  font-size: 14px;
  line-height: 1.4;
  font-weight: 700;
}

.xxx-form-panel-title::after {
  content: '';
  flex: 1;
  height: 1px;
  background: linear-gradient(90deg, rgba(203, 213, 225, 0.78), rgba(226, 232, 240, 0));
  transform: translateY(1px);
}

.xxx-form-note + .xxx-form-panel,
.xxx-form-panel + .xxx-form-panel {
  padding-top: 18px;
  border-top: 1px solid rgba(226, 232, 240, 0.92);
}
```

### 8.4 必填标识
- 使用浅色 `a-tag` 表示 `必填`，不使用默认红色星号
- 标签放在字段名称右侧，通过 `.xxx-form-label` 容器包裹
- 推荐样式：
  - 浅蓝描边：`border: 1px solid rgba(191, 219, 254, 0.72)`
  - 浅蓝背景：`background: rgba(239, 246, 255, 0.96)`
  - 蓝色文字：`color: #2563eb`
  - 小尺寸：`font-size: 11px; line-height: 18px`

**label 容器与 tag 的 CSS 参考：**
```css
.xxx-form-label {
  display: inline-flex;
  align-items: center;
  gap: 8px;
  color: #0f172a;
}

.xxx-form-required-tag {
  margin-inline-end: 0;
  border: 1px solid rgba(191, 219, 254, 0.72);
  background: rgba(239, 246, 255, 0.96);
  color: #2563eb;
  font-size: 11px;
  line-height: 18px;
}
```

**同时需要设置表单项 label 的基础样式：**
```css
.xxx-form :deep(.ant-form-item-label > label) {
  color: #0f172a;
  font-size: 13px;
  font-weight: 700;
}

.xxx-form :deep(.ant-form-item) {
  margin-bottom: 14px;
}
```

### 8.5 输入控件
- 控件背景保持透明，不使用厚重填充底
- 边框使用轻中性色
- Focus 时只加浅蓝描边和轻光晕
- 禁止把输入区再包一层独立玻璃卡片

**CSS 参考（弹窗内表单输入控件）：**
```css
/* 默认态 — 透明背景 + 浅描边 */
.xxx-form :deep(.ant-input),
.xxx-form :deep(.ant-select-selector),
.xxx-form :deep(.ant-input-affix-wrapper) {
  background: transparent !important;
  border-color: rgba(203, 213, 225, 0.88) !important;
  box-shadow: none !important;
}

/* Hover — 浅蓝描边 */
.xxx-form :deep(.ant-input:hover),
.xxx-form :deep(.ant-select:not(.ant-select-disabled):hover .ant-select-selector),
.xxx-form :deep(.ant-input-affix-wrapper:hover) {
  border-color: rgba(96, 165, 250, 0.48) !important;
}

/* Focus — 浅蓝描边 + 轻光晕 */
.xxx-form :deep(.ant-input:focus),
.xxx-form :deep(.ant-input-focused),
.xxx-form :deep(.ant-input-affix-wrapper-focused),
.xxx-form :deep(.ant-select-focused .ant-select-selector) {
  background: transparent !important;
  border-color: rgba(59, 130, 246, 0.56) !important;
  box-shadow: 0 0 0 3px rgba(59, 130, 246, 0.12) !important;
}

/* Disabled — 灰色文字 */
.xxx-form :deep(.ant-select-disabled .ant-select-selector),
.xxx-form :deep(.ant-input[disabled]) {
  background: transparent !important;
  color: #94a3b8 !important;
}

/* Placeholder */
.xxx-form :deep(.ant-select-selection-placeholder),
.xxx-form :deep(.ant-input::placeholder) {
  color: #94a3b8;
}
```

> 注意：以上是弹窗表单内的控件样式（透明底）。全页表单（如 ApplicationForm `surface="plain"`）使用另一套毛玻璃控件样式，见 `ApplicationForm.vue`。

## 9. 响应式规范
- 桌面端维持 `760` 宽度基线
- 小屏下沿用 `a-modal` 默认收缩行为
- 当内容区变窄时：
  - 标题栏允许保持左右结构
  - 上下文区两列降为一列
- 不允许为单个弹窗做全屏化特例

## 10. 禁用项
- 禁止恢复默认 `a-modal` footer 操作区
- 禁止右上角默认关闭图标与标题栏保存按钮并存
- 禁止使用深色遮蔽、重黑背景或高不透明度蒙层
- 禁止把整个表单区做成多层嵌套白卡
- 禁止在说明块、上下文块、分区块上叠加厚重背景与阴影
- 禁止遮蔽覆盖侧边导航

## 11. 推荐模板
```vue
<a-modal
  :open="visible"
  :width="760"
  :closable="false"
  :footer="null"
  :destroy-on-close="true"
  :after-close="handleAfterClose"
  :mask-style="modalMaskStyle"
  :wrap-props="modalWrapProps"
  wrap-class-name="xxx-form-modal-wrap"
  @cancel="handleCancel"
>
  <template #title>
    <div class="xxx-form-modal-titlebar">
      <span class="xxx-form-modal-title">{{ mode === 'create' ? '新增' : '编辑' }}</span>
      <a-button class="application-toolbar-action-btn xxx-form-modal-save-btn" :loading="submitting" @click="handleSubmit">
        保存
      </a-button>
    </div>
  </template>

  <a-form ref="formRef" :model="formState" layout="vertical" :required-mark="false" class="xxx-form">
    <!-- 说明块：创建/编辑模式均展示，可切换文案 -->
    <div class="xxx-form-note">
      {{ mode === 'create' ? '创建说明文案，描述表单用途。' : '编辑态只开放允许修改的字段，其余字段保持只读。' }}
    </div>

    <div v-if="mode === 'edit'" class="xxx-form-panel xxx-form-panel--context">
      <div class="xxx-form-panel-title">当前配置</div>
      <div class="xxx-form-context">
        <!-- readonly items -->
      </div>
    </div>

    <div class="xxx-form-panel">
      <div class="xxx-form-panel-title">{{ mode === 'create' ? '基础配置' : '可编辑配置' }}</div>

      <!-- 表单项示例：使用 #label 插槽 + 必填 tag -->
      <a-form-item name="field_name">
        <template #label>
          <span class="xxx-form-label">
            字段名
            <a-tag class="xxx-form-required-tag">必填</a-tag>
          </span>
        </template>
        <a-input v-model:value="formState.field_name" placeholder="请输入" />
      </a-form-item>
    </div>
  </a-form>
</a-modal>
```
