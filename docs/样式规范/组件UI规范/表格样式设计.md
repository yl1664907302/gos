# 表格样式设计

## 1. 目的
- 固化当前“执行器参数页”这套管理型表格的 UI 基线。
- 明确表格在页面中的位置、表头颜色、行背景、固定操作列、字段收敛方式和详情查看规则。
- 避免列表页重新回到“厚白卡包表格 + 列过多 + 表格内展示全部低频字段”的旧做法。

## 2. 适用范围
- 前端技术栈：`Vue 3 + Ant Design Vue`
- 适用页面：参数管理页、配置管理页、资源管理页、映射管理页等“以字段扫描和查看详情为主”的管理列表
- 当前基线页面：
  - `/Users/lingyunxieqing/Desktop/gos/frontend/src/views/component/ExecutorParamManagementView.vue`

## 3. 与旧表格规范的关系
- 本文档是对 `/Users/lingyunxieqing/Desktop/gos/docs/样式规范/组件UI规范/05-表格.md` 的补充。
- `05-表格.md` 继续定义表格基础技术约束，例如 `a-table`、`row-key`、横向滚动、状态列和分页。
- 本文档只描述当前这套“管理型亮底表格”的视觉语言和字段展示策略。

## 4. 页面中的表格位置

### 4.1 承载方式
- 表格直接放在主内容区中，不额外包一层厚重白色卡片背景。
- 外层容器只保留结构语义和 `padding: 0`，视觉重点交给表格本体。
- 这样可以让顶部查询操作区和下方数据区更连贯，避免页面出现“查询卡 + 表格卡”双层壳体。

### 4.2 与标题区关系
- 表格紧接页面头部查询区之后出现。
- 标题区只承载页面标题和查询操作，不在表格上方再插入摘要卡片、概览条或重复说明块。

## 5. 表格视觉基线

### 5.1 表格本体
- 表格整体背景透明，由 `ant-table-container` 自己承担边框和圆角。
- 推荐基线：
  - 容器圆角：`18px`
  - 边框：浅灰蓝描边
  - 外层无额外白卡阴影

### 5.2 表头
- 表头使用深石板蓝渐变，不使用默认浅灰表头。
- 当前基线颜色：
  - `linear-gradient(180deg, #243247, #1f2a3d)`
- 表头文字使用浅色高对比文本。
- 推荐规格：
  - 字号：`12px`
  - 字重：`700`
  - 轻微字距拉开

### 5.3 表体行
- 每行单元格使用轻透明白底，不做强分块卡片化。
- 行与行之间通过浅描边分隔，不依赖重阴影。
- Hover 只做轻微提亮，不做夸张颜色跳变。
- 当前基线：
  - 默认行背景：`rgba(255, 255, 255, 0.64)`
  - Hover 背景：`rgba(248, 250, 252, 0.92)`

## 6. 固定操作列

### 6.1 位置
- `操作`列固定在最右侧。
- 在横向滚动时，操作区始终保持可见，避免用户还要拖到最右边才能进行查看或编辑。

### 6.2 背景规则
- 固定列不能透明。
- 因为页面底层已经取消表格外层白卡，如果固定列透明，滚动时会出现透底和视觉脏边。
- 当前基线：
  - 固定列默认背景：接近纯白的实体底色
  - 固定列表头：与普通表头保持同一深色渐变
  - Hover 时固定列继续保持实体底色，不跟随透明

## 7. 列展示策略

### 7.1 列表只展示核心字段
- 列表页只保留“扫一眼能判断和定位”的高频字段。
- 当前执行器参数页基线展示：
  - `应用`
  - `所属执行器`
  - `真实参数名`
  - `平台标准 Key`
  - `参数类型`
  - `状态`
  - `必填`
  - `默认值`
  - `参数描述`
  - `排序`
  - `操作`

### 7.2 低频字段不在表格里铺开
- 以下低频元信息从主表格移除：
  - `展示`
  - `可编辑`
  - `来源`
  - `更新时间`
- 这些字段不适合常驻列表，否则列表会变宽、主信息被稀释、操作扫描效率下降。

### 7.3 完整信息通过查看获得
- 表格行内保留 `查看` 操作。
- 用户需要完整信息时，统一进入详情抽屉或详情面板查看。
- 列表负责快速扫描，详情负责完整呈现，这两者不要混在同一层里。

## 8. 单元格内容规范
- 状态列统一使用 `a-tag`
- 布尔字段如果在列表中出现，统一转成 `是 / 否`
- 空值统一显示 `-`
- 长文本字段使用 `ellipsis`
- 操作列统一使用 `a-button type="link" size="small"`

## 9. 横向滚动与宽度
- 管理型表格必须开启横向滚动，不强行把所有列压进一屏。
- 当前执行器参数页基线：
  - `:scroll="{ x: 1640 }"`
- 列宽应结合业务字段重要性分配：
  - 身份字段更宽
  - 状态、布尔、排序等短字段更窄
  - 操作列单独固定

## 10. 分页与空态

### 10.1 分页
- 分页固定放在表格下方右侧。
- 推荐保留：
  - `show-size-changer`
  - `show-quick-jumper`
  - 总数文案，例如 `共 X 条`

### 10.2 空态
- 空态直接出现在表格区域，不额外起一块说明卡片。
- 没有上下文时给明确指引，例如：
  - `请使用右上角搜索应用 / 应用Key / 平台Key`

## 11. 禁用项
- 禁止重新给表格外面套一层厚白卡背景。
- 禁止把低频字段全部重新塞回主列表。
- 禁止让固定操作列透明。
- 禁止恢复默认浅灰表头风格。
- 禁止用大量边框、阴影和色块把表格做成“卡片表格拼盘”。
