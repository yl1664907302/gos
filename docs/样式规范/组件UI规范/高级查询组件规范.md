# 高级查询组件规范

## 1. 目的
- 统一页面筛选区中"高级查询"面板的布局、交互与样式规范。
- 高级查询面板承载多条件筛选，与便捷查询按钮形成互补：便捷查询负责常用条件的快速切换，高级查询负责长尾条件的精细筛选。
- 明确"修改不立即生效、点击查询后统一提交"的交互模型，避免频繁自动查询打断使用节奏。

## 2. 适用范围
- 前端技术栈：`Vue 3 + Ant Design Vue`
- 适用场景：列表页筛选区中需要展开/收起的多条件查询面板
- 不适用于：放大镜浮层搜索、顶栏轻量筛选下拉框

## 4. 总体布局

### 4.1 面板结构
高级查询面板采用"操作栏 + 字段网格"的两段式布局：

```
┌─────────────────────────────────────────────────────┐
│  高级条件需点击"查询"后生效 · 有未生效的条件   [查询] [重置]  │  ← 操作栏
├─────────────────────────────────────────────────────┤
│  检索词（span 3）    │  应用（span 3）  │  操作类型  │  触发方式  │  创建时间  │  发起人  │  ← 字段网格
└─────────────────────────────────────────────────────┘
```

- 操作栏位于面板顶部，与字段网格之间用分隔线隔开。
- 字段网格使用 CSS Grid，6 列等宽布局，支持自动换行。

### 4.2 与便捷查询的关系
- 高级查询面板位于便捷查询按钮行下方，通过"高级检索"按钮切换展开/收起。
- 便捷查询（状态、环境）修改后立即生效；高级查询字段修改后需点击"查询"才生效。
- 两者共享同一个筛选卡片容器（`.filter-card`），不使用额外嵌套。

## 5. 操作栏规范

### 5.1 布局
- 操作栏使用 flex 布局，左对齐提示文字，右对齐操作按钮。
- 操作栏底部使用 `1px solid rgba(148, 163, 184, 0.12)` 分隔线与字段区隔开。

```css
.filter-actions-row {
  display: flex;
  align-items: center;
  justify-content: space-between;
  gap: 12px;
  margin-bottom: 16px;
  padding-bottom: 14px;
  border-bottom: 1px solid rgba(148, 163, 184, 0.12);
}
```

### 5.2 提示文字
- 默认提示：`高级条件需点击"查询"后生效`
- 有未生效条件时追加：`· 有未生效的条件`
- 样式：`color: var(--color-text-muted); font-size: 12px;`

```css
.filter-actions-hint {
  color: var(--color-text-muted);
  font-size: 12px;
  line-height: 1.5;
}
```

### 5.3 操作按钮
- 查询按钮：使用 `release-toolbar-action-btn--primary` 变体。
- 重置按钮：使用 `release-toolbar-action-btn--ghost` 变体。
- 按钮组右对齐，间距 `10px`。
- 查询按钮不绑定 loading 状态（避免与列表加载状态耦合导致按钮不可点击）。

```css
.filter-actions-buttons {
  display: flex;
  align-items: center;
  gap: 10px;
}
```

## 6. 字段网格规范

### 6.1 网格布局
- 使用 CSS Grid，`grid-template-columns: repeat(6, minmax(0, 1fr))`。
- 列间距：`gap: 14px 16px`（行间距 14px，列间距 16px）。
- 长字段（检索词、应用）使用 `grid-column: span 3` 占半行。
- 短字段（操作类型、触发方式、创建时间、发起人）各占 1 列。

```css
.filter-grid {
  display: grid;
  grid-template-columns: repeat(6, minmax(0, 1fr));
  gap: 14px 16px;
}

.filter-grid-item--keyword {
  grid-column: span 3;
}

.filter-grid-item--app {
  grid-column: span 3;
}
```

### 6.2 表单项规范
- 使用 `a-form layout="vertical"` 垂直布局。
- Label 样式：`font-size: 12px; color: var(--color-text-secondary);`
- 各控件宽度 100%，由网格列宽约束。

```css
.filter-grid :deep(.ant-form-item) {
  margin-bottom: 0;
}

.filter-grid-item :deep(.ant-form-item-label) {
  padding-bottom: 6px;
}

.filter-grid-item :deep(.ant-form-item-label > label) {
  color: var(--color-text-secondary);
  font-size: 12px;
}
```

### 6.3 控件类型
| 字段类型 | 控件 | 说明 |
|---------|------|------|
| 关键词 | `a-input` | 支持 `allow-clear`，回车触发查询 |
| 枚举选择 | `a-select` | 支持 `allow-clear`，`placeholder="全部"` |
| 模糊搜索枚举 | `a-select` + `show-search` | 如应用选择器 |
| 时间范围 | `a-range-picker` | `value-format="YYYY-MM-DD"` |
| 文本模糊 | `a-input` | 如发起人，回车触发查询 |

### 6.4 回车查询
- 关键词输入框和文本模糊输入框绑定 `@keydown.enter.prevent="handleSearch"`。
- 回车等同于点击"查询"按钮。

## 7. 面板容器样式

```css
.filter-advanced-panel {
  margin-top: 18px;
  padding: 18px;
  border-radius: 18px;
  border: 1px solid rgba(148, 163, 184, 0.16);
  background: transparent;
  box-shadow: none;
}
```

- 面板使用透明背景，不添加额外阴影或实色底。
- 圆角 `18px`，与筛选卡片整体风格一致。

## 8. 交互规范

### 8.1 生效模型
- 高级查询字段修改后进入暂存状态（`filters`），不立即生效。
- 只有点击"查询"按钮后，暂存值同步到生效查询（`activeQuery`），列表才正式刷新。
- 便捷查询（状态、环境）修改后立即生效，不受此规则约束。

### 8.2 未生效状态提示
- 当 `filters` 中的高级字段与 `activeQuery` 不一致时，提示文字追加"有未生效的条件"。
- 用户可通过提示文字感知当前是否有待提交的修改。

### 8.3 展开/收起
- 通过页面头部的"高级检索"按钮切换展开/收起。
- 如果当前存在已生效的高级筛选条件，面板默认展开。

### 8.4 重置
- 点击"重置"清空所有筛选条件（含便捷查询和高级查询），关闭面板，立即刷新列表。

## 9. 响应式

### 9.1 中等屏幕（≤1024px）
- 网格降为 2 列：`grid-template-columns: repeat(2, minmax(0, 1fr))`。
- 长字段（检索词、应用）降为 `grid-column: span 1`。

### 9.2 小屏（≤768px）
- 网格降为 1 列：`grid-template-columns: 1fr`。
- 操作栏改为纵向排列（提示文字在上，按钮在下）。

```css
@media (max-width: 768px) {
  .filter-actions-row {
    flex-direction: column;
    align-items: flex-start;
    gap: 10px;
  }
}
```

## 10. 推荐实现模板

```vue
<template>
  <div v-if="showAdvancedSearch" class="filter-advanced-panel">
    <div class="filter-actions-row">
      <div class="filter-actions-hint">

        <template v-if="hasPendingChanges">
          · 有未生效的条件

      </div>
      <div class="filter-actions-buttons">
        <a-button
          class="release-toolbar-action-btn release-toolbar-action-btn--primary"
          @click="handleSearch"

          查询
        </a-button>
        <a-button
          class="release-toolbar-action-btn release-toolbar-action-btn--ghost"

        >
          重置
        </a-button>
      </div>
    </div>
    <a-form layout="vertical" class="filter-grid">
      <a-form-item label="检索词" class="filter-grid-item filter-grid-item--keyword">
        <a-input
          v-model:value="filters.keyword"

          placeholder="全部"
          :options="applicationOptions"
        />
      </a-form-item>
      <a-form-item label="操作类型" class="filter-grid-item">
        <a-select
          v-model:value="filters.operation_type"

          placeholder="全部"
          :options="operationTypeOptions"
        />
      </a-form-item>
      <a-form-item label="触发方式" class="filter-grid-item">
        <a-select
          v-model:value="filters.trigger_type"
          allow-clear
          placeholder="全部"
          :options="triggerTypeOptions"
        />
      </a-form-item>
      <a-form-item label="创建时间" class="filter-grid-item">
        <a-range-picker
          v-model:value="filters.created_at_range"
          value-format="YYYY-MM-DD"
          format="YYYY-MM-DD"

        />

        <a-input

        />
      </a-form-item>

  </div>

```

## 11. 禁止项
- 禁止高级查询字段修改后自动触发查询，必须通过"查询"按钮手动提交。
- 禁止查询按钮绑定 loading 状态与列表加载耦合，避免快捷筛选或翻页操作导致按钮不可点击。
- 禁止为高级面板添加实色背景或厚重阴影，统一使用透明底。
- 禁止在高级面板内嵌套额外的卡片容器。
- 禁止将"查询"和"重置"按钮放在字段网格底部（必须放在顶部操作栏）。
